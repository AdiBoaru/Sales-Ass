# NX-81 — Tool agent: book_appointment + Google Calendar
**Owner:** S · **Faza:** P2 · **Zi/Ord:** după G7 (toolset agent) · **Branch:** `feat/NX-81-book-appointment` · **Complexitate:** M · **Estimare:** 4h

## Goal
Agentul (mini) capătă tool-ul `book_appointment(service_name, preferred_datetime, contact_info)`
care creează o programare reală în `appointments` (booked) și, dacă integrarea e configurată,
o oglindește în Google Calendar (păstrând event id-ul în `external_ref`). Deblochează
verticalele salon și HVAC: clientul cere o programare în chat, botul o scrie determinist
(scoped pe `business_id` + `contact_id`), fără ca operatorul să atingă un calendar manual.
Calendar-ul e BEST-EFFORT: dacă integrarea nu e gata / pică, programarea rămâne scrisă în DB
cu `external_ref IS NULL` (= „de sincronizat ulterior"), iar botul confirmă oricum slotul.

## Business Context
Pentru salon/HVAC, vânzarea = o programare, nu un coș. Fără acest tool, fiecare „vreau o
programare marți la 14" cade pe handoff (operator uman). Cu el, botul închide bucla 24/7,
scade handoff-urile și pune programarea direct în calendarul firmei. Calendar-ul deconectat
NU trebuie să blocheze rezervarea — operatorul vede în dashboard rândurile cu `external_ref`
nul și le sincronizează când integrarea e gata (degradare grațioasă, principiul 6).

## Technical Description

### Starea reală vs țintă
- `appointments` EXISTĂ în schema_v2 (`id, business_id, contact_id, conversation_id,
  service_name, starts_at, ends_at, status check('booked'|'confirmed'|'cancelled'|'no_show'|
  'done'), external_ref, notes, created_at`), cu index `idx_appointments_business_time
  (business_id, starts_at)`. `bot_runtime` are deja `select/insert/update` + RLS
  `business_id = current_business_id()` pe ea (003) → ZERO DDL.
- Framework de tool-uri EXISTĂ (`src/tools/base.py`): contract uniform
  `async def tool(ctx, deps, args: dict) -> ToolResult`, `@register("nume")`, `run_tool`
  prinde excepțiile → `ToolResult(ok=False)` (nu rupe turul). Toolset PER RUTĂ în
  `enabled_tools(business, route)`: azi `_SALES_TOOLS` + `_ORDER_TOOLS`. Adăugăm tool-ul.
- Agentul (`src/worker/stages/agent.py`) rulează DOAR pe `Route.SALES`/`Route.ORDER` și
  cheamă `tool_schemas(enabled_tools(...))`. Pentru salon/HVAC, intenția de programare cade
  azi pe ruta `sales` → expunem `book_appointment` pe toolsetul de vânzare, GATED pe vertical
  (o rută `book` dedicată în triaj = [PROPUNERE], out of scope aici).
- NU există încă integrare Google Calendar și NU există `proactive/scheduler.py` (doar
  `__init__.py`) → clientul de Calendar e NOU, izolat, injectat ca dependență (ca `MetaClient`).

### Tool-ul (cod determinist, scoped pe ctx — NU pe argumentele modelului)
`src/tools/booking_tools.py`:
```python
class BookAppointmentArgs(BaseModel):
    service_name: str = Field(min_length=1, max_length=120)
    preferred_datetime: str = Field(min_length=1)   # ISO-8601 sau text → parsat în cod
    notes: str | None = Field(default=None, max_length=500)
    # contact_info: NU vine de la model. Contactul e ctx.contact (din channel_identities) —
    # modelul nu „alege" pe cine programezi (izolare). Telefonul nu intră în appointments (P12).

@register("book_appointment")
async def book_appointment_tool(ctx, deps, args) -> ToolResult:
    a = BookAppointmentArgs(**args)
    starts_at = parse_local_dt(a.preferred_datetime, ctx.business.timezone)  # cod pur, fără LLM
    if starts_at is None:
        return ToolResult(ok=False, error="bad_datetime",
                          llm_view="Nu am înțeles data/ora. Cere clientului zi + oră clară.")
    if starts_at <= datetime.now(UTC):
        return ToolResult(ok=False, error="past",
                          llm_view="Data e în trecut. Cere un slot viitor.")
    dur = appt_duration_min(ctx.business, a.service_name)   # din business.settings, default 60
    ends_at = starts_at + timedelta(minutes=dur)

    appt = await create_appointment(
        deps.conn, ctx.business.id, ctx.contact.id, ctx.conversation_id,
        service_name=a.service_name, starts_at=starts_at, ends_at=ends_at, notes=a.notes,
    )   # INSERT appointments (status='booked', external_ref=NULL); întoarce {id,...}

    synced = False
    if deps.calendar is not None:                # integrare configurată → best-effort
        try:
            event_id = await deps.calendar.create_event(
                ctx.business.id, a.service_name, starts_at, ends_at,
            )
            await set_appointment_external_ref(deps.conn, ctx.business.id, appt["id"], event_id)
            synced = True
        except Exception:                        # Calendar jos → lăsăm external_ref NULL (sync ulterior)
            log.warning("book_appointment: sync Calendar eșuat → external_ref rămâne NULL")

    ctx.emit("appointment_booked", service=a.service_name,
             starts_at=starts_at.isoformat(), synced=synced)   # FĂRĂ telefon/nume (P12)
    when = starts_at.strftime("%d.%m %H:%M")
    return ToolResult(ok=True, llm_view=f"Programare creată: {a.service_name} pe {when}.")
```
- `parse_local_dt` = cod determinist (ISO + câteva pattern-uri RO „marți 14:00"/„mâine la 10");
  ambiguu → `ok=False` cu `llm_view` care cere clarificare (modelul reformulează, principiul 6,
  NU recovery agent). Parsarea NL nu cheamă LLM — e clasificare deterministă în cod.
- `ToolResult` NU pune `products`/`prices`/`links` → validatorul (stagiul 8) nu e atins (nu-s
  prețuri/linkuri de produs). Ora din `llm_view` e grounded prin construcție (scrisă de cod).

### Clientul de Calendar (NOU, izolat, injectat — pattern MetaClient)
`src/integrations/google_calendar.py` — `GoogleCalendarClient(http: httpx.AsyncClient, ...)`
cu `async def create_event(business_id, service, starts_at, ends_at) -> str` (event id).
Credențialele vin din `channels.credentials_ref` / secret manager (NU secrete în DB, NU în
loguri). Se atașează la `PipelineDeps.calendar` (default `None` → integrare dezactivată).
`deps.calendar is None` → tool-ul scrie DOAR în `appointments` (propunerea din audit: „sync
ulterior"), `external_ref` rămâne NULL = marcaj de nesincronizat.

### Câmpuri TurnContext scrise aici
NICIUNUL nou. Tool-ul rulează în interiorul stagiului Agent; setează DOAR `ctx.events` (via
`ctx.emit`). `ctx.reply` rămâne owned de Agent/Validator (modelul compune confirmarea din
`llm_view`). `ctx.retrieval` NU e atins (nu-s produse).

## Principii CLAUDE.md aplicabile
- **P2 (LLM doar triaj+agent):** tool-ul e cod DETERMINIST; e chemat de agent (mini) prin
  `run_tool_loop`. Parsarea datei NU e LLM. Niciun apel nou de model.
- **P5 (un singur punct de ieșire):** tool-ul NU trimite mesaje; confirmarea iese tot prin
  Agent → Sender → outbox. Calendar = side-effect, nu canal de răspuns.
- **P6 (niciodată tăcere / degradare):** Calendar jos sau lipsă → programarea rămâne în DB,
  botul confirmă oricum; dată ambiguă → cere clarificare, nu tace.
- **P7 (business_id pe tot):** `create_appointment` / `set_appointment_external_ref` au
  `business_id = $1`; `contact_id` din `ctx.contact` (NU din args). RLS = plasa.
- **P12 (PII într-un loc):** `appointments` n-are telefon; `contact_info` nu se persistă acolo
  (identitatea e `channel_identities`). Evenimentele logă serviciu/oră/`synced`, FĂRĂ nume/telefon.

## Test Cases
**Happy Path:**
1. args `{service_name:"tuns", preferred_datetime:"2026-07-01T14:00"}`, `deps.calendar=None`
   → `create_appointment` apelat cu `business_id`/`contact_id` din ctx, `ok=True`, event
   `appointment_booked {synced:False}` emis.
2. Aceleași args, `deps.calendar` fake (întoarce `"evt_123"`) → `set_appointment_external_ref`
   apelat cu `"evt_123"`, `synced=True`.

**Edge Cases:**
1. `preferred_datetime` text RO („mâine la 10") parsat de `parse_local_dt` în timezone-ul
   businessului → slot corect; sau, dacă parserul nu-l prinde, `ok=False bad_datetime` (fără DB).
2. Vertical `beauty` → `book_appointment` ABSENT din `enabled_tools` (tool indisponibil agentului).
3. `notes` peste 500 char → Pydantic respinge ÎNAINTE de orice DB (`run_tool` → `ok=False`).

**Failure Cases:**
1. `deps.conn` la INSERT aruncă → `run_tool` prinde → `ToolResult(ok=False)`, turul continuă,
   nicio excepție propagată (test cu conn mock care ridică).
2. `deps.calendar.create_event` aruncă (Calendar 500/timeout) → fail-soft: programarea persistă,
   `external_ref` NULL, `appointment_booked {synced:False}`, `ok=True` (test).
3. Dată în trecut → `ok=False error="past"`, `create_appointment` NEAPELAT (test).

> Teste fără apeluri reale: `deps.conn` = mock asyncpg (assert pe SQL/argumente),
> `deps.calendar` = obiect fake cu `create_event` programabil (return / raise). ZERO Calendar/DB
> real, ZERO OpenAI în CI (tool-ul nu cheamă LLM).
