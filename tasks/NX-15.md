# NX-15 — Moderation gate inbound
**Owner:** S · **Faza:** P1-O1 · **Zi/Ord:** după G5a (gates) · **Branch:** `feat/NX-15-moderation-gate` · **Complexitate:** S/M · **Estimare:** 3h

## Goal
Pune o poartă de moderare în Gates (stagiul 3), ÎNAINTE de triaj/agent: un mesaj
toxic/abuziv e prins de endpointul de moderation (gratuit la OpenAI), primește un
răspuns neutru și NU ajunge la niciun apel LLM scump (triaj/agent). Repetarea
(3 flag-uri în 24h) trece contactul pe abuse blocklist (`contacts.is_blocked`),
iar un contact blocat e tăcut din start. Protejează costul și reputația.

## Business Context
Un bot public pe WhatsApp/Telegram primește și mesaje abuzive/spam. Fără gate, fiecare
ajunge la triaj+agent (cost + risc de răspuns nepotrivit). Moderation e gratuit la OpenAI
și rapid → filtrăm înainte de orice cheltuială. Blocklist-ul oprește abuzul repetat (m19
din diagrama v4) fără intervenție umană.

## Technical Description

### Plasare în Gates (ordinea porților, fiecare cu early-exit)
`gates_stage` (stagiul 3) primește porțile noi, în ordine:
```
1. bot_active=False        → halt_silent (kill-switch, existent)
2. contact.is_blocked      → halt_silent("contact_blocked")        [NOU — enforce blocklist]
3. handoff activ           → halt_silent (existent)
4. moderation flagged      → răspuns neutru + early-exit            [NOU — poarta principală]
5. risc (pattern)          → request_human + mesaj tranziție (existent)
```
Moderarea vine DUPĂ handoff (dacă un om e pe conversație, oricum tăcem) și ÎNAINTE de
risc (un mesaj abuziv → răspuns neutru, NU escaladare la om — nu premiezi abuzul).

### Apelul de moderation (prin adaptorul unic)
`src.agent.llm` capătă o metodă `moderate(text) -> ModerationResult` (singurul loc care
vorbește cu OpenAI, principiul 2 — moderation NU e generare, e un clasificator de
siguranță, ca embed):
```python
@dataclass(frozen=True)
class ModerationResult:
    flagged: bool
    categories: list[str]   # categoriile True (ex. ["harassment", "hate"])

async def moderate(self, text, *, model=None) -> ModerationResult:
    resp = await self._client.moderations.create(model=model or self.model_moderation, input=text)
    r = resp.results[0]
    data = r.categories.model_dump()
    return ModerationResult(flagged=r.flagged, categories=sorted(k for k, v in data.items() if v))
```
`model_moderation` din settings (default `omni-moderation-latest`).

### Poarta de moderare (în gates)
```python
async def _moderation_blocked(ctx, deps) -> bool:
    s = get_settings()
    if not s.moderation_enabled or deps.llm is None:   # fără cheie → skip (fail-open)
        return False
    body = (ctx.message.body or "").strip()
    if not body:
        return False
    try:
        res = await deps.llm.moderate(body)
    except Exception:                                   # API jos → fail-OPEN (nu bloca tot traficul)
        log.warning("moderation: apel eșuat → fail-open")
        return False
    if not res.flagged:
        return False
    ctx.emit("message_moderated", categories=res.categories)   # FĂRĂ corpul mesajului (P12)
    await _record_flag_and_maybe_block(ctx, deps)
    ctx.set_reply(NEUTRAL_MSG, cacheable=False)         # răspuns neutru, ne-cache-uit
    return True
```
**Fail-open** deliberat: dacă moderation pică (rețea/API), lăsăm mesajul să treacă normal —
indisponibilitatea moderării nu trebuie să tacă tot botul (principiul 6). E best-effort
safety, nu o poartă dură.

### Contor de flag-uri (Redis, NU analytics)
`analytics_events` e append-only fără SELECT din runtime (003) → contorul „3/24h" trăiește
în Redis (pattern „contor Redis" din CLAUDE.md stagiul 2):
```python
async def _record_flag_and_maybe_block(ctx, deps):
    if deps.redis is None:
        return
    key = f"modflags:{ctx.business.id}:{ctx.contact.id}"
    count = await deps.redis.incr(key)
    if count == 1:
        await deps.redis.expire(key, 86400)            # fereastra de 24h
    if count >= get_settings().moderation_block_threshold:   # default 3
        await block_contact(deps.conn, ctx.business.id, ctx.contact.id)
        ctx.emit("contact_blocked", flag_count=count)
```
`block_contact(conn, business_id, contact_id)` = `update contacts set is_blocked = true
where business_id = $1 and id = $2` (bot_runtime are UPDATE pe contacts). Best-effort:
un eșec Redis/DB nu rupe răspunsul neutru (deja setat).

### Câmpuri TurnContext scrise aici
`ctx.reply` (răspuns neutru) + `ctx.halt` (via `halt_silent` pe contact blocat) — owner
Gates, ca și acum. Niciun câmp nou.

## Principii CLAUDE.md aplicabile

## Test Cases
**Happy Path:**
1. `moderate` mock → `flagged=True` → `ctx.reply` neutral, `message_moderated` emis,
   triaj-mock (monkeypatch pe stage următor) neatins.
2. `moderate` mock → `flagged=False` → `ctx.reply is None`, pipeline continuă.

**Edge Cases:**
1. Body gol/None → moderation nu apelează API (skip), trece mai departe.
2. `deps.llm=None` (fără cheie) → moderation sărit; mesaj normal trece, risc încă detectat.
3. Contact `is_blocked=True` → halt înainte de orice apel de moderation.

**Failure Cases:**
1. `moderate` aruncă (API/rețea) → fail-open: fără reply de moderation, pipeline continuă,
   nicio excepție propagată.
2. `deps.redis=None` la flag → răspuns neutru setat oricum; contorul/blocklist sărit (best-effort).
3. Al N-lea flag cu `incr` care aruncă → răspunsul neutru rămâne (eroarea de contor e prinsă).

> Teste fără apeluri reale: `LLMClient.moderate` cu `AsyncOpenAI` mock (rezultat fabricat);
> `gates_stage` cu `deps.llm`/`deps.redis` fake. ZERO apeluri reale în CI.
