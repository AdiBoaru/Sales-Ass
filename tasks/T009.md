# T009 — docker-compose dev
**Owner:** J · **Faza:** MVP · **Zi:** Z1 · **Branch:** `chore/T009-compose-dev` · **Complexitate:** S · **Estimare:** 2h

## Goal
Mediul de dev pornit cu o comandă: Redis + webhook (hot reload) + worker. Postgres NU e local — e Supabase remote, ca să lucrați amândoi pe aceleași date demo.

## Business Context
Junior la 4h/zi nu are timp de pierdut cu „mediul meu e stricat". Compose = mediu identic pentru amândoi.

## Technical Description
`docker-compose.yml` cu servicii: `redis` (redis:7-alpine, healthcheck redis-cli ping), `webhook` (build local, `uvicorn src.webhook.app:app --reload --host 0.0.0.0`, volume `./src:/app/src` pt reload, port 8000, env_file .env, depends_on redis healthy), `worker` (același build, `python -m src.worker.consumer`, volume identic, depends_on redis). Notă: webhook/worker vor crăpa la start până există T040/T060 — acceptat; healthcheck-ul pe redis e verde de acum.

## Principii CLAUDE.md aplicabile
Stack-ul din CLAUDE.md: Redis Streams + FastAPI + worker separat — compose-ul reflectă exact separarea webhook/worker (P1: pipeline liniar are nevoie de consumer separat de API).

## Implementation Steps
1. Scrie compose-ul cu cele 3 servicii.
2. `docker compose up` → redis healthy.
3. Documentează în README (T019) comanda.
4. PR + merge.
