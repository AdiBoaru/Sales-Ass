# NX-155 - Pilot Data Pack: catalog, FAQ, aliasuri, embeddings, product URLs

**Owner:** TBD · **Faza:** P0 · **Track/Wave:** Pilot Readiness · **Branch:** `feat/NX-155-pilot-data-pack` · **Complexitate:** M · **Estimare:** 1-2 zile

## Goal

Asigura ca tenantul de pilot are date suficient de bune pentru ca raspunsurile web sa para inteligente: produse reale, imagini, linkuri, preturi, FAQ-uri, aliasuri si embeddings. Acesta este P0-ul de business: fara ground truth bun, agentul poate fi arhitectural corect si totusi sa para slab.

## Business Context

Pilotul nu evalueaza `agent.py`, Redis sau RLS. Evalueaza daca asistentul recomanda produse bune, raspunde la intrebarile uzuale si trimite clientul spre actiune. Documentul de pilot readiness pune data readiness primul: catalog, FAQ si aliasuri coerente inainte de refactoruri mari.

## Scope

1. Audit live DB pentru tenantul de demo/pilot:
   - produse active;
   - imagine reala in `product_images.url`;
   - `product_url` real;
   - `price` si moneda;
   - `list_price`, daca exista reduceri;
   - rating/review_count, daca frontendul le afiseaza;
   - `ai_summary` / details;
   - categorii si atribute utile pentru cautare;
   - embeddings generate.

2. FAQ si aliasuri:
   - intrebarile de livrare/retur/plata/program/garantie;
   - aliasuri pentru intentii frecvente;
   - aliasuri pentru categorii si sinonime comerciale.

3. Reproductibilitate:
   - export sau seed oficial pentru data pack;
   - documentatie pentru refacerea tenantului;
   - script/check care spune daca tenantul este pilot-ready.

## Implementation Steps

1. Creeaza script/raport `scripts/audit_pilot_data.py` sau echivalent existent extins.
2. Genereaza raport per business: counts + lista de goluri.
3. Completeaza/backfill `product_url`, FAQ, aliasuri si embeddings unde lipsesc.
4. Adauga export read-only `scripts/export_pilot_data_pack.py` pentru snapshot reproductibil.
5. Adauga `docs/PILOT-DATA-PACK.md` cu pasii de audit, seed/export si rerulare.
6. Adauga un smoke test sau command check: `scripts/audit_pilot_data.py --business <business_id>`.
