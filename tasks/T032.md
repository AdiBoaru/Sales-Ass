# T032 — 002: indexurile lipsă
**Owner:** S · **Faza:** MVP · **Zi:** Z1-2 · **Branch:** `feat/T032-sql-indexes` · **Complexitate:** S · **Estimare:** 1h

## Goal
Indexurile pentru TOATE query-urile din hot path, scrise acum când le poți gândi la rece, nu în ziua în care p95 sare la 4 secunde. Fiecare index de mai jos corespunde unui query concret din pipeline.

## Business Context
Latența percepută a botului = jumătate din impresia de calitate. Postgres fără indexuri pe pattern-urile reale = botul „gândește" vizibil.

## Technical Description
```sql
-- istoricul conversației (T045: ultimele 8):
create index if not exists idx_messages_conv_time
  on conv.messages(conversation_id, created_at desc);
-- lookup FAQ alias (T078):
create index if not exists idx_faq_aliases_lookup
  on catalog.faq_aliases(business_id, normalized_alias);
-- cache semantic (T080, P1 dar indexul acum):
create index if not exists idx_rcache_hnsw
  on catalog.response_cache using hnsw (query_embedding vector_cosine_ops);
create index if not exists idx_rcache_expiry on catalog.response_cache(expires_at);
-- search produse (T111): filtrele dure
create index if not exists idx_products_filter
  on catalog.products(business_id, is_active, category);
create index if not exists idx_products_concerns
  on catalog.products using gin (concerns);
create index if not exists idx_variants_product
  on catalog.product_variants(product_id) where is_active;
-- dedupe (T063) — verifică: unique-ul din 001 acoperă deja lookup-ul? dacă nu:
create unique index if not exists idx_dedupe
  on conv.inbound_dedupe(business_id, channel, provider_message_id);
-- taxonomy (T130, citită per business la build de prompt):
create index if not exists idx_taxonomy_business on catalog.taxonomy(business_id, kind);
```
Pentru fiecare: rulează EXPLAIN pe query-ul țintă și confirmă Index Scan. La 221 produse Postgres poate alege Seq Scan (corect la volum mic) — verifică cu `set enable_seqscan=off` că indexul MERGE, nu că e ales.
