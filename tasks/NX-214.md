# NX-214 — Rollout gradual cu kill-switches

**Owner:** Claude (build) + Adi (aprobare trepte) · **Faza:** 11 (Quality Overhaul) · **Status:** DRAFT — nu se implementează până la review dependențe + DoD · **Complexitate:** M
**Parent plan:** [docs/QUALITY-OVERHAUL-2026.md](../docs/QUALITY-OVERHAUL-2026.md) · **Depinde de:** NX-211, NX-213 (fluxul nou complet + în buget)
**Decizii ADR aplicabile:** D13, D14

## Goal
Migrarea în producție fără să riscăm clienții: nicio schimbare big-bang, totul reversibil.

## Scope
1. **Trepte:** shadow (fluxul nou rulează, clientul primește vechiul) → trafic intern →
   demo tenant → 5% conversații → 20% → 50% → 100%. Bucketing stabil per conversație
   (nu per mesaj), izolat per business_id.
2. **La fiecare treaptă se compară old-vs-new:** calitate (eșantion judecat) · latență · cost ·
   fallback rate · no-result rate · click pe produs · checkout · reclamații · handoff către om.
   Avansarea la treapta următoare = decizie explicită pe cifre, nu automată.
3. **Fiecare componentă majoră cu feature flag + kill-switch** (agent nou, search tool, critic,
   streaming); rollback la versiunea stabilă < 5 minute, documentat și TESTAT.

## Definition of Done (schiță — de rafinat la review)
- [ ] Mecanismul de bucketing + flags implementat și testat (inclusiv rollback-ul).
- [ ] Dashboard-ul de comparație old-vs-new per treaptă funcțional.
- [ ] 100% rollout atins SAU oprire documentată cu cauze; nicio treaptă sărită.
