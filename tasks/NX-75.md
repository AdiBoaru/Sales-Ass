# NX-75 — Media routing: STT voce (Whisper) în Gates
**Owner:** S · **Faza:** P2 · **Zi/Ord:** după NX-15 (gates) · **Branch:** `feat/NX-75-stt-voce-gates` · **Complexitate:** M · **Estimare:** 4h

## Goal
Pune transcrierea vocalelor în Gates (stagiul 3): un mesaj `content_type=audio/voice`
e descărcat de la canal (Meta/Telegram), transcris prin adaptorul unic (`llm.transcribe`,
Whisper) și textul rezultat e injectat în `ctx.message.body` ÎNAINTE de straturi /
triaj / agent. Acum `content_type` e plumbed peste tot (parser → envelope → processor →
`InboundMessage`) dar NICIUN stagiu nu-l citește: o voce ajunge `body=None` la `fallback_stage`
și primește „n-am înțeles" — botul e surd la voce. STT-ul o face vizibilă restului pipeline-ului,
care rămâne agnostic de modalitate.

## Business Context
Pe WhatsApp în RO o parte mare din clienți trimit notă vocală în loc de text (mai rapid pe telefon).
Fără STT, fiecare voce = un „n-am înțeles" → client frustrat + tur irosit + posibil handoff inutil.
Whisper transformă vocea în exact aceeași cale ca textul (search produs, FAQ, comandă), deci
deblochează un segment de trafic care altfel se pierde. Cost mic, marginal, plafonat (vezi Cost).

## Technical Description

### Stare curentă (verificat în cod)
- `webhook/meta.py::_extract_body` setează `media_id` pentru tipurile media (`audio` inclus) și
  întoarce `caption` ca body (la audio = `None`). Envelope-ul neutru duce `content_type` + `media_id`.
- `processor.handle_turn` construiește `InboundMessage(content_type=..., media_ref=event["media_id"])`
  și-l scrie și în `messages` (`media_ref`). Deci `content_type`/`media_ref` ajung intacte în `ctx`.
- `gates_stage` NU are nicio ramură pe `content_type` → voce → `body=None` → `fallback_stage`.
- `LLMClient` (`src/agent/llm.py`) are deja `moderate` și `embed` ca excepții de
  clasificator/utility prin adaptorul unic. NU are `transcribe`.
- NICIUN client de canal nu poate descărca media: `MetaClient` are doar `send_text`;
  `TelegramClient` are `get_updates/send_*/get_me`, fără `getFile`/download.
- `telegram/poller.py::_to_event` IGNORĂ azi mesajele media-fără-text (`"text" not in msg → None`),
  deci pe Telegram vocea nici nu ajunge la worker. WhatsApp e canalul real al acestui task; Telegram
  e extins doar cât să nu mai arunce vocea (vezi Out of Scope pentru ce NU acoperim pe TG).

### Adaptorul unic — `LLMClient.transcribe` (excepție în spiritul embed/moderate)
LLM se cheamă doar din triaj+agent (P2). STT NU e generare — e un transcriptor/clasificator de
siguranță-utilitate, exact categoria stabilită pentru `embed`/`moderate`. Se adaugă în `src/agent/llm.py`,
NU se cheamă direct din alt cod:
```python
async def transcribe(self, audio: bytes, *, filename: str = "audio.ogg",
                     model: str | None = None, language: str | None = None) -> str:
    """STT (Whisper) — transcriptor, NU generare (P2, ca embed/moderate). Folosit de Gates
    (NX-75). Ridică la eroare de API — caller-ul (gate) prinde și degradează (body rămâne None)."""
    resp = await self._client.audio.transcriptions.create(
        model=model or self.model_stt,
        file=(filename, audio),
        language=language,          # hint din ctx.language ('ro'/'hu'/'en') → acuratețe mai bună
    )
    return (resp.text or "").strip()
```
`model_stt` din settings (default `whisper-1`), adăugat în `__init__`/`get_llm` ca `model_moderation`.

### Descărcarea media — o margine de canal (NU în pipeline)
Descărcarea e transport de canal → trăiește la margine, ca `ChannelSender`. Definim un protocol
`MediaFetcher` în `src/channels/base.py` (lângă `ChannelSender`):
```python
@runtime_checkable
class MediaFetcher(Protocol):
    async def fetch_media(self, account_id: str, media_id: str) -> tuple[bytes, str]: ...
    # întoarce (bytes, filename_cu_extensie) — filename pt mime hint la Whisper
```
- **WhatsApp** (`MetaClient.fetch_media`): flux Graph în 2 pași — `GET /{media_id}` (cu Bearer) →
  `url` + `mime_type` → `GET url` (tot cu Bearer, host lookaside) → bytes. Extensia din mime
  (`audio/ogg` → `.ogg`). Cap dur de mărime (ex. 16MB Meta) → la depășire ridică → fail soft în gate.
- **Telegram** (`TelegramClient.fetch_media`): `getFile(file_id)` → `file_path` →
  `GET {base}/file/bot{token}/{file_path}` → bytes. (`account_id` informativ; tokenul e în URL.)
Un `MediaFetcherRegistry` (`channel_kind → MediaFetcher`), populat ca `build_registry` din dispatcher,
e injectat în `PipelineDeps.media` (nou câmp, `MediaFetcherRegistry | None`). Gate-ul cere fetcher-ul
după `ctx`-ul de canal — zero cod de canal în gate.

> NB transport: `MetaClient`/`TelegramClient` capătă acces la `httpx.AsyncClient` (deja injectat la
> dispatcher; pentru worker se construiește un registry propriu la bootstrap-ul worker-ului, ca la
> dispatcher). Tokenul Meta = `settings.meta_access_token`; descărcarea media cere același token ca trimisul.

### Poarta de transcriere (în `gates_stage`)
Plasată ÎNAINTE de moderation/risc (textul transcris trebuie moderat + verificat de risc ca orice
text). După kill-switch/blocklist/handoff (dacă tăcem, nu cheltuim Whisper degeaba):
```python
_AUDIO_TYPES = ("audio", "voice")  # voice = nota vocală WhatsApp; audio = fișier

async def _transcribe_voice(ctx: TurnContext, deps: PipelineDeps) -> None:
    s = get_settings()
    if (not s.stt_enabled or deps.llm is None or deps.media is None
            or ctx.message.content_type not in _AUDIO_TYPES):
        return
    if ctx.message.body:                       # are deja caption/text → nu re-transcriem
        return
    media_id = ctx.message.media_ref
    if not media_id:
        ctx.set_reply(VOICE_FALLBACK_MSG, cacheable=False)   # P6: nimic de descărcat
        return
    fetcher = deps.media.get(ctx.channel_kind)   # margine de canal
    if fetcher is None:
        ctx.set_reply(VOICE_FALLBACK_MSG, cacheable=False)
        return
    try:
        audio, filename = await fetcher.fetch_media(ctx.channel_account_id, media_id)
        text = await deps.llm.transcribe(audio, filename=filename, language=ctx.language)
    except Exception as e:  # noqa: BLE001 — download/STT jos → degradare grațioasă, NU tăcere
        log.warning("stt: transcriere eșuată (%s)", type(e).__name__)  # FĂRĂ media_ref/PII
        ctx.set_reply(VOICE_FALLBACK_MSG, cacheable=False)
        ctx.emit("voice_transcription_failed")
        return
    if not text:
        ctx.set_reply(VOICE_FALLBACK_MSG, cacheable=False)   # voce goală/zgomot
        return
    ctx.message.body = text                     # injectăm textul → pipeline continuă normal
    ctx.emit("voice_transcribed", chars=len(text))   # FĂRĂ textul (poate conține PII de adresă)
```
`VOICE_FALLBACK_MSG` = „N-am putut asculta nota vocală 🙂 Poți să-mi scrii pe scurt ce cauți?".
Integrare în `gates_stage` (ordinea reală): `1 bot_active → 2 blocklist → 3 handoff →
**4 _transcribe_voice (NOU)** → 5 rate limit → 6 moderation → 7 risc`. Important: `_transcribe_voice`
poate seta `ctx.reply` (early-exit pe eșec) SAU doar muta `ctx.message.body` (pipeline-ul continuă).

### Câmpuri TurnContext scrise aici
- `ctx.message.body` — INJECTAT cu transcrierea (singurul stagiu care rescrie body; e tot Gates,
  ownership-ul de modalitate stă aici, nu apare un al doilea writer pe alt câmp).
- `ctx.reply` (via `set_reply`) — pe eșec (fallback grațios), owner Gates ca acum.
Nicio scriere pe `route`/`retrieval`/`state` aici.

### Context de canal pe TurnContext
Gate-ul are nevoie de `channel_kind` + `channel_account_id` ca să ceară fetcher-ul corect. Azi NU
sunt pe `ctx` (doar în `event` din processor). Le propagăm pe `InboundMessage` (sau `TurnContext`):
adăugăm `InboundMessage.channel_kind` + `InboundMessage.channel_account_id` (umplute în processor
din `event`). Mic, fără ownership nou — tot owner processor (ca restul `InboundMessage`).

## Principii CLAUDE.md aplicabile
- **P1 (pipeline liniar):** STT injectează body în Gates → restul stagiilor rămân identice, fără
  ramuri de modalitate. Nicio săritură înapoi.
- **P2 (LLM doar triaj+agent):** `transcribe` = transcriptor prin adaptorul unic, NU generare —
  excepție explicită în spiritul `embed`/`moderate`. Nu se cheamă Whisper din alt cod.
- **P5 / margini de canal:** descărcarea media e transport → `MediaFetcher` la margine (ca
  `ChannelSender`), zero cod de canal în pipeline/gate.
- **P6 (niciodată tăcere):** download/STT jos / voce goală → fallback grațios („scrie-mi pe scurt"),
  NU tăcere.
- **P7 (business_id pe tot):** nu se adaugă query nou (vezi DB). Fetch-ul de media e per (canal,
  media_id), izolat de `ctx` care e deja tenant-scoped.
- **P12 (PII):** logurile NU conțin `media_ref`, URL-uri de media (au token), bytes audio sau textul
  transcris (poate conține adresă/telefon dictate). Evenimentele duc DOAR `chars`/flag, nu textul.
  Token-ul Meta/Telegram nu se loghează (httpx pe WARNING, ca în dispatcher/poller).

- **Telegram complet pentru voce** (download + STT e2e pe TG) — poller-ul doar nu mai aruncă vocea;

- **Persistarea transcrierii** în `messages.body`/coloană nouă (audit pe ce-a spus userul vocal).

- **Cache pe transcrieri** (audio identic → skip Whisper) — micro-optimizare, ulterior.

## Test Cases
**Happy Path:**
1. `content_type="voice"`, `media_ref="m1"`; fetcher mock → `(b"...", "v.ogg")`; `transcribe` mock →
   „caut o crema hidratanta" → `ctx.message.body == "caut o crema hidratanta"`, `voice_transcribed`
   {chars} emis, `ctx.reply is None` (pipeline continuă).
2. `MetaClient.fetch_media` cu `httpx.MockTransport`: `GET /m1` → `{"url": "...", "mime_type":
   "audio/ogg"}`, apoi `GET url` → bytes → întoarce `(bytes, "*.ogg")`; Bearer prezent pe ambele.

**Edge Cases:**
1. `content_type="text"` (sau orice non-audio) → `_transcribe_voice` iese imediat, zero apeluri.
2. Voce fără `media_ref` (envelope corupt) → fallback, fără apel fetch/STT.
3. Voce cu `body` (caption) deja setat → nu re-transcrie, body păstrat.
4. `deps.media.get(channel_kind)` → None (canal fără fetcher înregistrat) → fallback.

**Failure Cases:**
1. `fetcher.fetch_media` ridică (HTTP 404/timeout) → fallback + `voice_transcription_failed`, nicio excepție.
2. `llm.transcribe` ridică (OpenAI down) → fallback grațios.
3. `transcribe` → `""` (zgomot / liniște) → fallback (nu trimite text gol la triaj).
4. `deps.llm=None` (fără cheie OpenAI) → voce → fallback, NU tăcere (P6).

> Teste fără apeluri reale: `LLMClient.transcribe` cu `AsyncOpenAI` mock; `MetaClient.fetch_media`
> cu `httpx.MockTransport`; `gates_stage` cu `deps.llm`/`deps.media` fake. ZERO apeluri reale în CI.

## Cost
STT Whisper la OpenAI: **~$0.006 / minut** de audio. Notele vocale tipice pe WhatsApp ~10-20s →
~$0.0015-0.002 / voce. Pe 1000 mesaje, dacă ~20% sunt vocale (200 vocale × ~15s = ~50 min) →
**~$0.30 / 1000 mesaje**. Marginal față de agent (mini). Ce limitează costul:
- `stt_enabled` (kill-switch global) + STT doar pe `content_type` audio (nu pe text).
- `stt_max_bytes` cap pe download → vocale lungi/uriașe refuzate (fail soft), nu transcrise.
- Cost guard zilnic G2c rămâne plasa de buget (peste plafon → `llm=None` → voce → fallback, fără STT).
- Skip dacă există deja caption (nu plătim STT degeaba).
