## Goal
Limba devine parte din cheia de lookup pe toate sursele de răspuns fără LLM. Repară bug-ul latent găsit la audit: cache semantic care face match pe embedding cross-limbă și servește răspuns RO unui client maghiar.

## Business Context
Târgu Mureș + clientela HU = realitatea primilor clienți. Un răspuns în limba greșită arată neprofesionist exact acolo unde botul trebuie să pară impecabil.

## Technical Description
```sql
alter table catalog.faq add column language text not null default 'ro'
  check (language in ('ro','hu','en'));
alter table catalog.response_cache add column language text not null default 'ro'
  check (language in ('ro','hu','en'));
alter table catalog.clarification_templates add column language text not null default 'ro'
  check (language in ('ro','hu','en'));
-- cheile de lookup se actualizează:
-- faq_aliases se caută JOIN faq ON ... AND faq.language = $2
-- clarification_templates: unique(business_id, missing_field) devine unique(business_id, missing_field, language)
alter table catalog.clarification_templates
  drop constraint if exists clarification_templates_business_id_missing_field_key;
alter table catalog.clarification_templates
  add unique (business_id, missing_field, language);
```
Backfill: datele demo sunt RO → default-ul acoperă tot. Regula de cod (intră în CLAUDE.md P11, deja scrisă): orice query pe aceste tabele FĂRĂ language în WHERE = bug la review.
