# NX-201 — Pricing reconciliat + instrumentare per-stagiu + SLO provizoriu

**Owner:** Claude (build) / Codex (verify) · **Faza:** 0 (Quality Overhaul) · **Status:** DRAFT rev.2 (post-review Codex) · **Complexitate:** M
**Parent plan:** [docs/QUALITY-OVERHAUL-2026.md](../docs/QUALITY-OVERHAUL-2026.md)
**Depinde de:** — pentru felia A (pricing) și felia B (OTel LOCAL, fără exporter extern).
**NX-200 (politica de date) blochează DOAR felia C** (exporter către orice backend extern).
**Decizii ADR aplicabile:** D12, D14

## Goal
Să știm exact unde se duc cele ~14 secunde și cât costă REAL un tur — și să fixăm **SLO-ul
provizoriu** contra căruia se judecă prototipul (NX-210), ca gate-ul mare să nu depindă de
un card ulterior (NX-213).

## Scope
**Felia A — tarife (fără dependențe):** reconciliere `src/agent/pricing.py` cu factura OpenAI
reală → override `LLM_PRICING_JSON` în prod + corectare valori implicite; re-verificare
`usage_daily` + contoarele cost-guard după corecție; delta documentat.

**Felia B — spans locale (fără dependențe):** OTel ca strat neutru (D14), exporter LOCAL.
Stagii instrumentate: context loading · triage · fiecare apel LLM · embedding query · lexical ·
vector · fusion · rerank · evidence loading · validation · send · total tur. Atribute: tokeni,
cost/apel, cached tokens, model, nr. tool calls, nr. căutări, fallback-uri, retries.

**Felia C — exporter extern (blocată de NX-200):** doar după politica de date aprobată.

**Felia D — SLO provizoriu:** din baseline + țintele comerciale, se fixează pragurile contra
cărora se evaluează NX-210 (ex. „nu mai rău decât baseline pe facts; mediană sales sub X").
Țintele FINALE rămân în NX-213.

## Out of Scope
Orice optimizare (NX-213). Aici doar măsurăm + fixăm SLO-ul provizoriu.

## Definition of Done
- [ ] Tarife reconciliate cu factura; delta documentat; cost-guard re-verificat.
- [ ] Spans emise pe toate stagiile; **zero PII în atribute** (test dedicat).
- [ ] Raport baseline în `reports/`: latență mediană/P90 per stagiu + cost median/P90 per tur,
      pe eșantion sim + live; top 3 consumatori de timp identificați cu cifre.
- [ ] SLO provizoriu documentat în ADR (consumat de NX-210).
- [ ] Instrumentarea nu schimbă comportamentul (zero regresii) + kill-switch pe tracing.
- [ ] `business_id` prezent în spans ca atribut server-owned; test izolare neatins.
- [ ] `ruff check` + `ruff format --check` + `pytest -x -q` verzi.
