# NX-51 — P0-B: Dedupe inbound în 2 straturi (Redis + tabel ne-partiționat)
**Owner:** S · **Faza:** MVP · **Zi:** Z2 (ÎNAINTE de POST /webhook) · **Branch:** feat/NX-51-dedupe · **Complexitate:** S · **Estimare:** 5h

## Goal
Garantează că un mesaj WhatsApp retrimis de Meta (retry la timeout/5xx) este procesat EXACT O DATĂ. Înlocuiește T063 și unique-ul nefuncțional de pe `messages`.

## Business Context
P0-B din audit: fără asta, primul timeout în producție = clientul final primește două răspunsuri identice + cost LLM dublu + state corupt de tururi paralele. Vizibil direct de clientul plătitor.

## Technical Description
De ce nu merge ce există: unique-ul `(business_id, provider_msg_id, created_at)` pe `messages` partiționat include obligatoriu cheia de partiționare; retry-ul Meta sosește cu alt `created_at` → ON CONFLICT nu se declanșează. Fix în 2 straturi:
1. **Strat rapid (webhook):** `SET dedupe:{business_id}:{provider_msg_id} 1 NX EX 172800` în Redis. Dacă răspunsul e nil (cheia exista) → ACK 200 și STOP (nu XADD).
2. **Plasă durabilă (worker):** tabel nou `public.inbound_dedupe` NE-partiționat; `INSERT ... ON CONFLICT (business_id, provider_msg_id) DO NOTHING RETURNING 1`. Dacă nu întoarce rând → skip turul (mesajul a fost deja procesat — acoperă restart Redis / pierdere AOF).

## Principii CLAUDE.md aplicabile
P2 (business_id peste tot — inclusiv în cheia de dedupe), P6 (niciodată tăcere ≠ niciodată dublu), P9 (idempotență cap-coadă).

## Implementation Steps
1. Migrare `0xx_inbound_dedupe.sql`: tabel `(business_id uuid NOT NULL, provider_msg_id text NOT NULL, first_seen timestamptz DEFAULT now(), PRIMARY KEY (business_id, provider_msg_id))` — FĂRĂ partiționare.
2. `src/webhook/dedupe.py`: `async def seen_before(redis, business_id, wamid) -> bool` cu SET NX EX 172800.
3. În handlerul POST: după validarea semnăturii și extragerea wamid → dacă `seen_before` → log `dedupe_hit_redis` + return 200.
4. În consumer (primul pas al turului): INSERT în `inbound_dedupe`; fără rând returnat → log `dedupe_hit_db` + ACK pe stream + STOP.
5. `src/jobs/cleanup_dedupe.py`: DELETE WHERE first_seen < now() - interval '48 hours' (rulat de scheduler, zilnic).
6. Eveniment `analytics_events`: `inbound_deduped` cu stratul care a prins (redis|db).
