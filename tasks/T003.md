# T003 — PR template + CODEOWNERS
**Owner:** J · **Faza:** MVP · **Zi:** Z1 · **Branch:** `chore/T003-pr-template` · **Complexitate:** XS · **Estimare:** 0.5h

## Goal
Standardizează PR-urile (ce face / cum s-a testat / ID task) și forțează review-ul Seniorului pe căile periculoase: prompturi și SQL.

## Business Context
Promptul și schema sunt singurele locuri unde o schimbare „mică" poate degrada toate conversațiile tuturor clienților. CODEOWNERS face review-ul pe ele nenegociabil.

## Technical Description
`.github/PULL_REQUEST_TEMPLATE.md` cu secțiunile: **Task** (ID + link la tasks/TXXX.md), **Ce face**, **Cum s-a testat** (comenzi rulate), **Checklist** (DoD bifat, pytest local verde, fără secrete). `.github/CODEOWNERS`: `prompts/ @senior-github-user` și `docs/*.sql @senior-github-user`. În branch protection (T002) se activează „Require review from Code Owners".

## Principii CLAUDE.md aplicabile
P9 (taxonomia/prompturile sunt sensibile) — orice schimbare acolo trece prin doi oameni.

## Implementation Steps
1. Creează cele două fișiere cu username-urile reale.
2. PR + merge.
3. Seniorul activează „Require review from Code Owners" în rule-ul din T002.
4. Test: PR de junior care atinge `prompts/` cere obligatoriu review-ul Seniorului.

## Files To Modify

## Dependencies

## Out of Scope
- Reguli de review pe restul codului (rămâne review asimetric din CONTRIBUTING).

## Definition of Done
- [ ] PR nou vine precompletat cu template-ul
- [ ] PR care modifică `prompts/` sau `docs/*.sql` cere review de la Senior
- [ ] Username-urile din CODEOWNERS verificate (typo = regula nu se aplică silențios!)

## Test Cases
**Happy:** PR normal nu cere review obligatoriu. **Edge:** PR care atinge și prompts/ și src/ cere review. **Failure:** CODEOWNERS cu username greșit → GitHub arată warning în UI (verifică vizual).
