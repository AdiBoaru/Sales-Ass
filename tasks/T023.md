# T023 — 002: checkout_links + attribution pe orders
**Owner:** S · **Faza:** MVP · **Zi:** Z1-2 · **Branch:** `feat/T023-sql-attribution` · **Complexitate:** S · **Estimare:** 1h

## Goal
Structura buclei de bani: fiecare link de checkout generat de bot are un ref_code; comanda care vine cu acel ref devine revenue ATRIBUIT botului. Fără tabelul ăsta, dashboard-ul „botul a generat X RON" — argumentul principal de vânzare al retainer-ului — nu are sursă de date.

## Business Context
La renegocierea retainer-ului, cifra de revenue atribuit e singurul argument care nu se discută.

## Technical Description
```sql
create table conv.checkout_links (
  id uuid primary key default gen_random_uuid(),
  business_id uuid not null references core.businesses(id),
  conversation_id uuid not null,
  contact_id uuid not null references conv.contacts(id),
  ref_code text not null,                  -- scurt, URL-safe: 10 chars base62 din turn_id
  cart jsonb not null default '[]',        -- [{product_id, variant_id, qty, price_snapshot}]
  url text not null,
  clicked_at timestamptz,
  converted_order_id uuid,
  expires_at timestamptz,
  created_at timestamptz not null default now(),
  unique (business_id, ref_code)
);
alter table conv.orders
  add column attributed_checkout_link_id uuid references conv.checkout_links(id),
  add column attribution text not null default 'none'
    check (attribution in ('none','assisted','direct_bot'));
create index idx_orders_attributed on conv.orders(business_id, created_at)
  where attribution <> 'none';
```
`price_snapshot` în cart = prețul din momentul recomandării (dovadă la dispute). `assisted` = comanda a venit de la un contact cu conversație activă <72h dar fără ref (regulă aplicată în T189).
