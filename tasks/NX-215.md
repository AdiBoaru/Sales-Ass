# NX-215 — Ritual săptămânal de calitate (proces permanent, pornit din Faza 1)

**Owner:** Adi + Claude · **Faza:** 12 (Quality Overhaul) — dar PORNEȘTE odată cu NX-202/203 · **Status:** DRAFT — nu se implementează până la review dependențe + DoD · **Complexitate:** S (setup) + permanent
**Parent plan:** [docs/QUALITY-OVERHAUL-2026.md](../docs/QUALITY-OVERHAUL-2026.md) · **Depinde de:** NX-202, NX-203 (seturile); se îmbogățește pe parcurs
**Decizii ADR aplicabile:** D10, D14, D15

## Goal
Motorul care transformă produsul din „bun la lansare" în „mai bun în fiecare lună", pe date
reale — singura fază fără dată de final.

## Scope (bucla săptămânală)
1. Eșantion de conversații reale (redactate PII) → evaluare (rubrici: fapte · need-alignment ·
   calitatea întrebării · naturalețe RO — LLM-judge informativ + review uman pe regresii mari).
2. Gruparea erorilor pe clustere: produs corect neintrat în candidați · constrângere neextrasă ·
   evidence incomplet · recomandare generică · întrebare inutilă · răspuns lent · informație
   lipsă din catalog.
3. Top-3 cauze → fix (date / retrieval / prompt) → re-rulare golden set + retrieval benchmark →
   comparație cu săptămâna anterioară.
4. Review-ul batch al vocabularului viu (pipeline NX-208) — 15 min.
5. Metrici comerciale urmărite: click produs, checkout, response-rate la clarificări, handoff.

## Tooling
Harness-urile NX-202/203 + telemetria existentă (NX-162/163/164) + tracing NX-201.
Gate CI: regresie pe retrieval/validator blochează merge; scorul LLM-judge NU blochează singur.

## Definition of Done (schiță — de rafinat la review)
- [ ] Ritualul documentat (checklist repetabil) + primul ciclu complet rulat.
- [ ] Raport săptămânal standard (template) cu trend-uri.
- [ ] Dovada buclei închise: minim un fix derivat din clustere, cu îmbunătățire măsurată.
