# G7-1 — Agent tool-calling: framework + read core (search/details/compare)
**Owner:** S · **Faza:** P1 · **Zi/Ord:** Faza 1 din epicul „tool-uri agent" · **Branch:** `feat/G7-1-agent-tools` · **Complexitate:** L · **Estimare:** ~6h

## Goal
Convertește agentul (stagiul 7) din RAG (1 search + 1 apel) într-un agent cu **tool-calling
determinist** (cap DUR 3 apeluri), apoi livrează primele 3 tool-uri read-only:
`search_products` (wrap pe cel existent), `get_product_details` (cu rezumatele D3) și
`compare_products`. Agentul DECIDE ce tool cheamă; tool-urile sunt cod determinist scoped pe
`business_id`. Foundație pentru fazele de comerț/comenzi. Arhitectura completă:
[`docs/agent-tools-architecture.md`](agent-tools-architecture.md).

## Business Context
Agentul poate acum să intre în detaliile unui produs (rating + ce laudă clienții, din D3) și
să compare 2-3 produse la cerere — răspuns mai bogat, tot grounded pe catalog real. Fundația
de tool-calling deblochează apoi bucla de bani (cart/checkout, faza 2).

## Technical Description

### Framework (`src/tools/base.py`)
- `ToolResult(ok, products, llm_view, error)`: `products` = complete (pt `ctx.retrieval`/
  validator); `llm_view` = COMPACT (max 6×8, fără PII) — ce vede modelul.
- `TOOL_REGISTRY` (nume→fn), `register(name)`, `enabled_tools(business)` (Faza 1: cele 3
  read; per-business ulterior), `run_tool(ctx, deps, name, args)` (dispatch + try/except →
  `ToolResult(ok=False)` la eșec; tool eșuat NU rupe turul).
- Semnătură tool: `async def tool(ctx, deps, args: dict) -> ToolResult`. Argumente validate
  Pydantic ÎNAINTE de execuție. `business_id` din `ctx`, NU din argumentele modelului (izolare).

### Bucla (în adaptorul unic `src.agent.llm` — singurul loc OpenAI)
`run_tool_loop(system, user, tools, execute, max_steps=3)`:
- Apel `chat.completions.create(tools=..., tool_choice="auto")`. Dacă `tool_calls` → execută
  CONCURENT (`asyncio.gather`), adaugă mesajele `assistant`+`tool`, repetă. Fără tool_calls →
  text final. La atingerea `max_steps` → un ultim apel `tools=None` (text forțat).
- `execute(name, args) -> str` = callback dat de agent: rulează tool-ul (registry), acumulează
  produsele, emite `tool_call`, întoarce `llm_view` modelului. Formatul OpenAI (tool_calls/rol
  tool) stă DOAR în adaptor.

### Agent (`agent_stage` rescris)
- Doar `route='sales'`, `deps.llm` prezent. Construiește `tools = tool_schemas(enabled)`
  (prefix STATIC → prompt caching). Rulează `run_tool_loop` cu `execute` care acumulează
  `retrieved`.
- `ctx.retrieval = RetrievalResult(products=dedupe(retrieved)[:6], source="tools")`.
- Cu produse → `_finalize` (validează → 1 retry `complete` cu prețuri permise → fallback
  determinist) + carduri W1. Fără produse dar cu text → servește textul (clarify/chitchat).
  Fără nimic → mesaj de clarificare. Degradare grațioasă păstrată.

### Validator (stagiul 8, extins ușor)
Păstrează `_prices_ok` (preț ∈ retrieval) + adaugă **link check**: orice URL din reply ∈
`product_url`-urile retrievate (nu inventat). Numele de produs (fuzzy) = refinement ulterior.

### Tool-uri (`src/tools/catalog_tools.py`)
- `search_products(query, price_max?, limit=6)` → embed + `search_products_semantic`.
- `get_product_details(product_id)` → `get_products_by_ids` (preț, rating, summary+top_pros/
  cons+sentiment din D3).
- `compare_products(product_ids: 2-3)` → `get_products_by_ids` → vedere comparativă compactă.
- Query nou `get_products_by_ids(conn, business_id, ids, limit)` în `catalog.py` (`where
  business_id=$1 and id = any($2::uuid[])`).

### Scheme OpenAI (`src/agent/tool_definitions.py`)
JSON function schemas pt cele 3 + `tool_schemas(names)`. Strict (Structured Outputs) ca
argumentele să fie valide din construcție.

## Principii CLAUDE.md aplicabile
- **P2:** LLM doar în agent, prin adaptor; tool-urile = cod determinist.
- **P7:** fiecare query în tool are `business_id = ctx.business.id`; modelul NU primește

- **P8 (ref-uri):** `llm_view` compact (≤6×8), nu obiecte complete.
- **Validator (zero halucinații):** preț + link verificate; retry→fallback determinist.
- **P10:** runner/agent emit `tool_call {name, ok, latency}` fără PII.
- **MAX 3 tool calls:** cap DUR în cod (în adaptor).

## Test Cases
**Happy Path:**
1. FakeLLM: tool_call `search_products` → text final grounded → `ctx.reply` + carduri,
   `ctx.retrieval` populat, `tool_call`+`agent_recommended` emise.
2. `get_product_details` cu un produs mock → `llm_view` conține rating + „clienții laudă".
3. `compare_products([a,b])` → vedere cu ambele.

**Edge Cases:**
1. Adaptor `run_tool_loop` cu mock OpenAI: 3 pași cu tool_calls → al 4-lea forțează text
   (`tools=None`).
2. `compare_products` cu 1 id valid → `ok=False` + mesaj.
3. Model răspunde direct (fără tool_call) → text servit, fără carduri.

**Failure Cases:**
1. Tool inexistent („foo") → `run_tool` → `ok=False`, bucla continuă (nu aruncă).
2. Args invalide (`search_products` fără `query`) → Pydantic respinge → `ok=False`.
3. Validator pică pe textul final → retry `complete` → fallback determinist (prețuri reale).

> Teste FĂRĂ apeluri reale: `run_tool_loop` cu mock `AsyncOpenAI`; `agent_stage` cu FakeLLM
> care scriptează tool_calls+text; tool-urile cu conn/llm mock. ZERO OpenAI în CI.

## Cost
Tool-calling = ~2-4× apeluri mini/tur de vânzare (1 decizie + N pași + final, cap 3). Atenuat
de prompt caching (prefix static de scheme), cache semantic G5b (servește repetările), cap dur
3, doar `sales` → agent. Vezi `docs/agent-tools-architecture.md` §I.6. Cost guard + rate limit
(P0) într-un task separat — tool-calling le face urgente.
