# G5b-1 — Semantic cache: lookup (exact+semantic) + write-back gated (tier static)
**Owner:** S · **Faza:** P1 · **Zi/Ord:** după G5a · **Branch:** `feat/G5b-semantic-cache` · **Complexitate:** M · **Estimare:** 4h

## Goal
Stagiul 4 (strat gratuit): înainte de triaj/agent, întoarce din cache răspunsul la
query-uri repetate, fără apel LLM. Construiește scheletul precision-first din
`docs/semantic-cache-design.md`: canonicalize + router de volatilitate + L1 exact +
L2 semantic (prag conservator) + write-back gated async. **În G5b-1 servim DOAR
tierul `static`** (FAQ/întrebări generice, fără produse/prețuri); `dynamic`
(recomandări de produs) e rutat pe lângă cache până când G5b-2 aduce invalidarea.

## Business Context
Pârghia de cost a sistemului. Traficul de suport e Zipfian — un cap mic de întrebări
domină; L1 exact îl servește la O(1) și sigur. Scade apeluri LLM + latența, fără
risc de preț învechit (produsele nu se cache-uiesc încă). Foundație + instrumentare
ca să măsurăm UNDE e repetiția înainte să calibrăm.

## Technical Description

### Stagiul în pipeline
`cache_stage` se inserează ÎNAINTE de `triage_stage` (după `gates_stage` dacă G5a e
în main): `[gates?, cache_stage, triage_stage, agent_stage, fallback_stage]`. Un hit
setează `ctx.reply` → early-exit (sare triaj+agent). Rulează cod determinist; singurul
apel extern e `embed()` (text-embedding-3-small), NU un model de generare.

### Canonicalizare (determinist)
```python
def canonicalize(text: str) -> tuple[str, str]:
    s = unicodedata NFKD lower, fără diacritice, colaps spații, strip punctuație/filler
    return s, sha256(s).hexdigest()   # canonical_str, canonical_hash
```
Ridică hit-rate (paraphrase → același entry) + alimentează L1 exact prin hash.

### Router de volatilitate (determinist, fără triaj)
Cache-ul rulează ÎNAINTE de triaj → clasifică singur, pe pattern-uri:
```python
def classify_volatility(text) -> "static" | "dynamic" | "realtime":
    realtime: comandă/personal ("comanda", "awb", "unde e coletul", "a mea")  → bypass
    dynamic:  buget/număr+lei SAU produs-ish ("cremă", "preț la", brand)       → bypass (G5b-1)
    else:     static                                                            → eligibil
```
`dynamic` + `realtime` → cache-ul NU servește în v1 (return fără reply, pipeline-ul
continuă). `static` → L1 → L2.

### L1 exact (KV)
`exact_lookup(conn, business_id, locale, canonical_hash)` → `(business_id, locale,
canonical_hash)` cu `expires_at > now()`. Hit ⇒ `ctx.reply`, `ctx.from_cache=True`,
`touch_hit` (hit_count+1, last_hit_at), emit `cache_lookup{layer:'exact'}`. O(1), zero
false-positive.

### L2 semantic (pgvector)
Miss pe L1 → `embed(canonical_str)` → `semantic_lookup`:
```sql
select id, answer, 1 - (embedding <=> $3) as similarity
from semantic_cache
where business_id = $1 and locale = $2 and expires_at > now()
  and volatility_class = 'static'
order by embedding <=> $3
limit 1
```
Auto-accept DOAR dacă `similarity >= τ_high` (conservator, default 0.92; din config).
Hit ⇒ ca la L1, emit `cache_lookup{layer:'semantic', similarity}`. Sub prag ⇒ miss
(emit `cache_lookup{layer:'miss', similarity}`), pipeline continuă. **Fără gray-zone
verify în v1** — precizie peste recall (cârlig pt faza 2).

### Write-back gated, async (post-tur)
După ce pipeline-ul produce reply, în post-tur (ca extractorul de profil, best-effort,
NU blochează răspunsul): scrie entry DOAR dacă TOATE:
- `not ctx.from_cache` (nu re-scriem un hit),
- `classify_volatility(body) == 'static'`,
- `ctx.reply.products is None` (un răspuns cu produse = dynamic → exclus în v1),
- reply nu e clarify/refuz (fără `pending_question`; nu e fallback-ul generic),
- `len(reply.text)` rezonabil (nu gol).

Scrie `semantic_cache`: business_id, locale, query_norm=canonical_str, canonical_hash,
embedding, answer, volatility_class='static', embedding_model, quality_score (din
context — v1: 1.0 dacă a trecut gate-ul), expires_at = now()+TTL_static (config, default
7 zile), is_curated=false. Idempotent pe `(business_id, canonical_hash)` (upsert:
on conflict update answer+embedding+expires_at).

### Câmpuri TurnContext scrise aici
`ctx.reply` (pe hit) + `ctx.from_cache` (nou, bool, owner: cache_stage). Write-back-ul
nu scrie TurnContext (rulează post-tur, citește ctx).

## Principii CLAUDE.md aplicabile
- **P2 (LLM doar triaj+agent):** cache-ul folosește DOAR `embed()` (embeddings), NU
  generare. Canonicalize + classify sunt determinist.
- **P7 (business_id pe tot):** fiecare query are `business_id = $1`. **P11 (limba e
  parte din cheie):** lookup-ul include `locale`. RLS = plasa (NX-50/04).
- **P3 (un owner per câmp):** `ctx.from_cache` scris doar de cache_stage.
- **P12 (PII):** corpul query-ului NU se loghează; instrumentarea logă `layer`,
  `similarity`, `intent` — nu textul.
- **P6 (niciodată tăcere):** hit = reply; miss = pipeline continuă (nu tace).

## Test Cases
**Happy Path:**
1. `exact_lookup` hit pe canonical_hash → `ctx.reply` setat, `from_cache=True`,
   triaj-mock neatins.
2. `canonicalize("Care e POLITICA de retur??")` == `canonicalize("care e politica de
   retur")` (același hash).

**Edge Cases:**
1. `classify_volatility("vreau o cremă sub 80 lei")` == 'dynamic' → bypass (nu servește).
2. L2 candidat la `similarity = τ_high - 0.01` → miss (nu auto-accept sub prag).

**Failure Cases:**
1. `embed()` aruncă (fără cheie OpenAI) → cache_stage degradează grațios (miss, fără
   excepție), pipeline-ul continuă.
2. Write-back pe un reply cu `products` → NU scrie (produs = dynamic, exclus în v1).

> Teste fără apeluri reale: `canonicalize`/`classify_volatility` = unit pur;
> `cache_stage` cu `exact_lookup`/`semantic_lookup`/`embed` monkeypatch-uite; un test
> mic de integrare pe `upsert_entry`/`exact_lookup` (DB real). ZERO apeluri de generare.

## Cost
Generare: **ZERO**. Adaugă `embed(canonical)` per query static eligibil
(~$0.00001/query) + un query pgvector. Economisește triaj(nano)+agent(mini) pe fiecare
hit. Per 1000 mesaje: cost embed neglijabil; economia = hit_rate × (cost triaj+agent).
Măsurăm cu instrumentarea înainte să promitem un procent (review §10).
