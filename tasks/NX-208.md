# NX-208 — QuerySpec pe 3 reprezentări (shadow) + pipeline vocabular viu privacy-safe

**Owner:** Claude (build) / Codex (verify) · **Faza:** 5 (Quality Overhaul) · **Status:** BUILT (felia de înțelegere + contract + dovadă cifre; vocab-viu + disagreement-report deferred — vezi DoD) · **Complexitate:** M
**Rezultat măsurat:** Recall@20 0,667→0,806 · Top-6 0,667→0,833 · Forbidden@6 0,250→0,167 · 0 regresii. Detalii: [docs/NX-208-QUERY-UNDERSTANDING.md](../docs/NX-208-QUERY-UNDERSTANDING.md).
**Parent plan:** [docs/QUALITY-OVERHAUL-2026.md](../docs/QUALITY-OVERHAUL-2026.md) · **Depinde de:** NX-200 (D11 decisă) · **Reconciliază:** NX-185 (existent)
**Decizii ADR aplicabile:** D6, D7, D10, D11

## Goal
QuerySpec devine contractul unic al „ce cere clientul" — cu raw query-ul PĂSTRAT — și
vocabularul viu primește un pipeline curat, cu aprobare umană.

## Scope
1. **Extindere contract NX-185 cu separarea Runtime/Safe (D6):**
   - **`RuntimeQuerySpec`** — `raw_query` + `normalized_query` + `canonical_constraints[]`
     (hard/soft per NX-185). Trăiește **DOAR în memoria turului**; NU se scrie în DB, log,
     trace sau analytics (raw query-ul poate conține nume/telefon/adresă/date medicale).
   - **`SafeQuerySpec`** — `canonical_constraints[]` + metadate normalizate, FĂRĂ `raw_query`,
     fără PII. **Singurul** obiect persistabil/telemetrizabil.
   Căutarea folosește toate reprezentările în paralel; canonicalizarea adaugă precizie, nu înlocuiește.
2. **Reconciliere ownership (D11):** extracția în triaj rămâne DOAR shadow (comparație);
   ownership-ul țintă = agentul principal (F7). Enforcement NX-187/188 rămâne înghețat.
   Telemetria `query_spec_disagreement` (NX-185) se păstrează — e exact datele care arată
   unde tool-loop-ul slăbește constrângeri.
3. **concern_map doar potriviri de mare încredere** („mă lucesc" → oily/matte). Expresiile noi
   NU intră automat.
4. **Pipeline vocabular viu (D10, mecanism NOU** — unmet_query nu conține text brut):
   expresie client → redactare PII → normalizare → agregare → prag minim de frecvență →
   review uman (batch săptămânal, 15 min) → intrare aprobată în concern_map/intent_aliases.

## Out of Scope
Enforcement QuerySpec (post-F7). Search tool-ul (NX-209).
