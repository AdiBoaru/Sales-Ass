# T017 — OpenAI: chei + limite de spend
**Owner:** S · **Faza:** MVP · **Zi:** Z1 · **Branch:** — · **Complexitate:** XS · **Estimare:** 0.5h

## Goal
Protecție financiară înainte de primul apel: chei separate dev/prod și plafoane hard la nivel de cont, ca un bug în loop să coste zeci de dolari, nu mii.

## Business Context
Cost guard-ul din cod (T176) e în P1 — până atunci, singura plasă e plafonul din contul OpenAI.

## Technical Description
platform.openai.com → două proiecte: `nativx-dev`, `nativx-prod`, cheie per proiect. Billing → Usage limits: hard limit lunar (ex. 50 USD dev / 200 USD prod la început), email alert la 50%/80%. Cheile în .env local și mai târziu în secrets pe VPS (T158). Verifică accesul la modelele din stack (gpt-5.4-mini, gpt-5.4-nano, text-embedding-3-small) cu un apel de test per model.

## Principii CLAUDE.md aplicabile
Anticipează cost guard-ul per business (T176) — plafonul de cont e stratul 0.

## Implementation Steps
1. Creează proiectele + cheile.
2. Setează limitele + alertele.
3. Apel de test pe fiecare model din stack.
4. Cheile în .env la ambii (dev key pentru junior).
