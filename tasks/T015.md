# T015 — Tunel local dev (cloudflared)
**Owner:** S · **Faza:** MVP · **Zi:** Z1 · **Branch:** — (tooling local, doar docs) · **Complexitate:** XS · **Estimare:** 1h

## Goal
Meta are nevoie de HTTPS public ca să livreze webhook-uri; tunelul expune localhost:8000 fără deploy. Dev-ul rămâne local, VPS-ul e doar staging/prod.

## Business Context
Ciclu de feedback de secunde (salvezi → uvicorn reload → mesaj test) în loc de deploy per schimbare.

## Technical Description
`cloudflared tunnel --url http://localhost:8000` (quick tunnel, URL nou la fiecare pornire — pentru MVP ok; tunel named cu domeniu fix e bonus P1). Alternativă dacă cloudflared face probleme: `ngrok http 8000`. URL-ul rezultat se pune în Meta webhook config (re-verify la fiecare schimbare de URL — de asta tunelul named merită în P1). Documentezi procedura în README.

## Principii CLAUDE.md aplicabile
— (tooling).

## Implementation Steps
1. Instalează cloudflared local (ambii).
2. Pornește tunelul, copiază URL-ul https.
3. Pune URL+/webhook în Meta config → verify (T014) verde.
4. Documentează în README: „la fiecare restart de tunel, re-verifică webhook-ul".
