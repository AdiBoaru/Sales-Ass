# T014 — Webhook verify endpoint (GET)
**Owner:** S · **Faza:** MVP · **Zi:** Z1 · **Branch:** `feat/T014-webhook-verify` · **Complexitate:** XS · **Estimare:** 0.5h

## Goal
Meta validează URL-ul de webhook printr-un GET cu challenge înainte să trimită orice eveniment. Endpoint-ul ăsta e poarta de intrare a întregului sistem.

## Business Context
Fără verify, zero mesaje de la Meta — totul în aval e mort.

## Technical Description
`GET /webhook` în FastAPI minimal (poate trăi înainte de T060 complet — un app schelet ajunge): citește query params `hub.mode`, `hub.verify_token`, `hub.challenge`. Dacă mode==subscribe și verify_token == META_VERIFY_TOKEN (din env; voi îl inventați — string random lung) → return PlainTextResponse(challenge, 200). Altfel → 403. Atenție: challenge-ul se întoarce ca text brut, NU JSON (Meta compară exact).

## Principii CLAUDE.md aplicabile
P5 în oglindă: ăsta e singurul punct de INTRARE (alături de POST-ul din T061).

## Implementation Steps
1. App FastAPI minimal în src/webhook/app.py (dacă nu există încă) + ruta GET.
2. Test unitar: token corect → challenge înapoi ca text; token greșit → 403.
3. Configurare în Meta App → WhatsApp → Configuration: URL-ul de tunel (T015) + verify token → Verify and Save (verde).
4. Subscribe la câmpul `messages`.
