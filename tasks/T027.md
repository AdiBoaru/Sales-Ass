# T027 — 002: messages provider_message_id + status
**Owner:** S · **Faza:** MVP · **Zi:** Z1-2 · **Branch:** `feat/T027-sql-msg-status` · **Complexitate:** XS · **Estimare:** 0.5h

## Goal
Mesajele outbound primesc identitatea lor Meta (wamid) și ciclul de viață (queued→sent→delivered→read / failed). Fără asta, webhook-ul de statusuri (T065) nu are pe ce să mapeze, iar „clientul a citit oferta?" nu are răspuns.

## Business Context
Read rate per conversație = semnal de lead scoring (P1) și de sănătate a botului (mesaje failed în masă = numărul are probleme).

## Technical Description
```sql
alter table conv.messages
  add column provider_message_id text,
  add column status text not null default 'received'
    check (status in ('received','queued','sent','delivered','read','failed'));
create index idx_messages_provider on conv.messages(business_id, provider_message_id)
  where provider_message_id is not null;
```
Semantica: inbound → status 'received' (default, neschimbat ulterior). Outbound → inserat cu 'queued' de sender (T097), trecut la 'sent' + wamid de dispatcher (T098), apoi delivered/read/failed de webhook status (T065). Index parțial: lookup-ul de status e mereu pe wamid non-null, inbound-urile nu poluează indexul.
