# NX-73 — Strat gratuit: alias lookup (intent_aliases / phrase_norm)
**Owner:** S · **Faza:** P1 · **Zi/Ord:** după G5b (cache) · **Branch:** `feat/NX-73-alias-lookup` · **Complexitate:** S · **Estimare:** 3.5h

## Goal
Adaugă stratul gratuit CEL MAI IEFTIN al pipeline-ului (stagiul 4, ÎNAINTE de cache
semantic și triaj): `phrase_norm(text)` → match EXACT în `intent_aliases`
(`status='approved'`, `business_id`). La hit → rută deterministă SAU răspuns FAQ +
early-exit, fără niciun apel LLM (nici măcar embed). Acum lipsește complet: nu există
query, nu există stagiu, nu e în `DEFAULT_STAGES` (`runner.py:80-87`). E lookup pe index
(`idx_aliases_lookup`), deci ~0 latență și $0 cost. Valoarea reală apare după ce
`intent_aliases` se populează din shadow mode (NX-93); pe demo e 0 acum.

## Business Context
40-60% din trafic ar trebui să se oprească în straturile gratuite, înainte de orice
LLM. Aliasurile sunt cea mai ieftină felie: fraze recurente exacte (ex. „program",
„livrare", „retur", „unde e comanda") rezolvate dintr-un index B-tree, zero token.
Fiecare hit aici economisește un apel nano (triaj) + eventual mini (agent). E mecanismul
prin care întrebările repetate devin gratuite pe măsură ce shadow mode-ul promovează
candidați în `approved`.

## Technical Description

### Plasare în pipeline (ÎNAINTE de cache și triaj)
`alias_stage` se inserează în `DEFAULT_STAGES` (`src/worker/runner.py`) între `language_stage`
și `cache_stage`:
```
gates → language → ALIAS (NOU) → cache → triage → agent → fallback
```
Motivul ordinii: aliasul e match EXACT pe index (mai ieftin și mai sigur decât embed-ul
semantic din cache), iar limba e deja refinată de `language_stage` (necesară pt răspunsul FAQ
locale-keyed — principiul 11). Early-exit pe `ctx.reply` (FAQ) SAU pe `ctx.route` deterministă
(rutare) preempt-ează cache + triaj.

### Normalizare (refolosește `cache/canonical.py`, NU duplica)
`phrase_norm` din schemă = „lowercased, fără diacritice". `canonicalize(text)` din
`src/cache/canonical.py` face EXACT asta (lowercase + NFKD fără diacritice + punctuație→spațiu
+ colaps spații + strip) și întoarce `(canonical_str, hash)`. Folosim `canonical_str` ca
`phrase_norm`. ATENȚIE: la populare (NX-93) candidații trebuie scriși cu ACEEAȘI funcție, altfel
match-ul exact ratează — documentat în Out of Scope.

### Query nou — `src/db/queries/aliases.py`
```python
async def lookup_alias(
    conn: asyncpg.Connection, business_id: str, phrase_norm: str
) -> dict | None:
    """Match EXACT pe alias aprobat. Folosește idx_aliases_lookup (business_id,
    phrase_norm) WHERE status='approved'. conn = tenant_conn (RLS plasă)."""
    row = await conn.fetchrow(
        """
        select id::text as id, target_kind, target_id::text as target_id, target_value
        from intent_aliases
        where business_id = $1
          and phrase_norm = $2
          and status = 'approved'
        order by created_at desc
        limit 1
        """,
        business_id,
        phrase_norm,
    )
    return dict(row) if row else None


async def get_faq_answer(
    conn: asyncpg.Connection, business_id: str, faq_id: str, locale: str
) -> str | None:
    """Răspunsul unui FAQ activ pe (business_id, id, locale). locale = parte din
    filtru (principiul 11): un FAQ în limba greșită NU e un hit."""
    row = await conn.fetchrow(
        """
        select answer from faqs
        where business_id = $1 and id = $2 and locale = $3 and is_active = true
        limit 1
        """,
        business_id,
        faq_id,
        locale,
    )
    return row["answer"] if row else None
```
`target_kind` are 4 valori (`faq|product|category|route`). Maparea în stagiu:
- **`faq`** → fetch `get_faq_answer(target_id, ctx.language)`; dacă există → `ctx.set_reply(answer)`
  (cacheabil — răspuns static reutilizabil) + early-exit. Dacă FAQ-ul lipsește în limba
  curentă → NU servim (miss grațios, lasă pipeline-ul mai departe).
- **`route`** → `target_value` e una din rutele `Route` (`simple|sales|order|handoff|clarify`);
  setăm `ctx.route = RouteDecision(route=...)` și lăsăm stagiile downstream (agent/order) să
  răspundă. NU setăm reply — doar scurtcircuităm triajul (un apel nano economisit).
- **`product` / `category`** → setăm `ctx.route = RouteDecision(route=Route.SALES,
  category_key=<slug>)` pentru a sări triajul și a alimenta agentul. Pentru `category`,
  `target_value` = slug-ul; pentru `product`, `target_id` = product_id (agentul îl preia prin
  retrieval). MVP: tratăm ambele ca rutare `sales` cu `category_key` când avem slug; altfel doar
  `route=sales`.

### Stagiul — `src/worker/stages/alias.py`
```python
async def alias_stage(ctx: TurnContext, deps: PipelineDeps) -> None:
    settings = get_settings()
    if not settings.alias_enabled:
        return
    body = (ctx.message.body or "").strip()
    if not body:
        return
    phrase_norm, _ = canonicalize(body)
    if not phrase_norm:
        return
    try:
        alias = await lookup_alias(deps.conn, ctx.business.id, phrase_norm)
        if alias is None:
            ctx.emit("alias_lookup", hit=False)
            return
        kind = alias["target_kind"]
        if kind == "faq":
            answer = await get_faq_answer(deps.conn, ctx.business.id,
                                          alias["target_id"], ctx.language)
            if answer is None:                       # FAQ lipsă în limba curentă → miss
                ctx.emit("alias_lookup", hit=False, target_kind=kind, reason="faq_locale_miss")
                return
            ctx.set_reply(answer)                    # cacheabil (static)
            ctx.emit("alias_lookup", hit=True, target_kind=kind)
        elif kind == "route":
            route = _coerce_route(alias["target_value"])   # None dacă invalid → miss
            if route is None:
                ctx.emit("alias_lookup", hit=False, target_kind=kind, reason="bad_route")
                return
            ctx.route = RouteDecision(route=route)
            ctx.emit("alias_lookup", hit=True, target_kind=kind, route=route.value)
        else:  # product | category → rutare sales (+ category_key dacă avem slug)
            ctx.route = RouteDecision(route=Route.SALES,
                                      category_key=alias.get("target_value"))
            ctx.emit("alias_lookup", hit=True, target_kind=kind)
    except Exception as e:  # noqa: BLE001 — strat gratuit best-effort → miss, nu rupe turul
        log.warning("alias: lookup eșuat (%s) → miss", type(e).__name__)
```
Best-effort identic cu cache-ul: orice eroare (migrare neaplicată, DB jos) → degradare la „miss",
NU rupe turul (principiul 6). ZERO LLM în acest stagiu (cod pur).

### Câmpuri TurnContext scrise aici
- `ctx.reply` (via `set_reply`) — pe hit FAQ; early-exit la Sender.
- `ctx.route` (RouteDecision) — pe hit route/product/category, scurtcircuitând triajul.
  > Conflict de owner: `ctx.route` e „owner Triaj" în contract. Aici stagiul de alias scrie
  > `ctx.route` ÎNAINTE ca triajul să ruleze, iar early-exit-ul pe reply/route opreste pipeline-ul
  > la agent fără a mai trece prin triaj — deci nu sunt doi scriitori simultani pe același tur.
  > Documentat explicit ca excepție de scurtcircuit (la fel ca „simple" din triaj care setează reply).

## Principii CLAUDE.md aplicabile
- **P2 (LLM doar triaj+agent):** alias_stage e COD PUR determinist, ZERO LLM (nici embed). E cel

## Test Cases
> Toate cu stub DB (fake `conn` care întoarce înregistrări fabricate). ZERO apeluri reale
> OpenAI/DB în CI. `deps.llm` poate fi None (alias_stage nu-l folosește).

**Happy Path:**
1. Alias `faq` aprobat pe `phrase_norm` egal cu `canonicalize(body)` + FAQ activ în `ctx.language`
   → `ctx.reply.text` = answer, early-exit; stub de `cache_stage` care aruncă NU e atins.
2. Alias `route='order'` → `ctx.route.route == Route.ORDER`, `ctx.reply is None`, triaj sărit.

**Edge Cases:**
1. Body cu diacritice/majuscule („Progrăm?") → `canonicalize` îl normalizează la `phrase_norm`
   stocat („program") → hit (dovedește refolosirea normalizării, nu duplicat).
2. Alias `category` cu `target_value` slug → `ctx.route = SALES` + `category_key=slug`; fără reply.
3. `alias_enabled=False` → stagiul nu apelează DB, pipeline-ul trece direct la cache.
4. Body gol/None → skip fără apel DB.

**Failure Cases:**
1. `lookup_alias` ridică excepție (DB jos / migrare neaplicată) → miss grațios, fără excepție,
   pipeline continuă (best-effort, principiul 6).
2. Alias `route` cu `target_value` invalid (ex. „buy") → `_coerce_route` întoarce None → miss,
   `alias_lookup{hit:false, reason:"bad_route"}` emis.
3. Alias `faq` dar FAQ-ul nu există în `ctx.language` → `get_faq_answer` None → miss
   (`reason:"faq_locale_miss"`), pipeline continuă la cache.

## Cost
**$0 LLM / 0 tokeni** — stagiu cod pur, fără apel OpenAI (nici embed). Per 1000 mesaje: doar
1-2 round-trip-uri DB pe index B-tree (`idx_aliases_lookup`), sub-milisecund fiecare. ECONOMISEȘTE
1 apel nano (triaj) + eventual 1 apel mini (agent) pe fiecare mesaj care lovește un alias — exact
opusul unui cost. Ce limitează costul: niciun cost de adăugat; deflection-rate-ul (deci economia)
e mărginit de câți aliasuri `approved` există (0 pe demo până la NX-93).
