# NX-168e-2 — Graful PDP: sections + ingredients + badges + reviews (seed complet)

**Owner:** S · **Faza:** P1 · **Track:** Demo Data Overhaul / Catalog v3 · **Branch:** `feat/NX-168e-2-pdp-graph` · **Complexitate:** M-L · **Estimare:** 2-3 zile
**Parent:** [tasks/NX-168e.md](NX-168e.md) (partea 2) · **Depinde de:** NX-168e-1 (catalog la contract v3, PR #222) mergeat

## Goal
Rescrie seed-ul ca să populeze **graful complet de PDP** al produsului — tabelele care există în schema_v2 dar sunt goale pentru catalogul demo: `product_sections`, `ingredients` + `product_ingredients`, `product_badges`, `reviews` individuale. Astea dau botului adâncimea de „detaliu de produs" (cum se folosește, avertizări, INCI normalizat, badge-uri de trust, social proof) — **fiecare tabel populat cu un consumator clar în bot** (regula anti-date-moarte).

## Business Context
168e-1 a adus faptele de recomandare (attributes) la contract. 168e-2 aduce **adâncimea PDP**: `get_product_details` extins (sections + INCI + badges + review-intel), filtru pe ingrediente normalizate, badge-uri pe card. Fără consumator, un tabel populat = date moarte → fiecare felie se leagă de o suprafață în bot.

## Technical Description
- **`product_sections`** (kind: usage/benefits/warnings/how_to): derivate din `attributes.usage` + `key_benefit` + `not_recommended_for` (avertizări), autorate acolo unde e nevoie. Poziționate.
- **`ingredients` + `product_ingredients`**: INCI normalizat din `attributes.key_ingredients` + `claim_provenance` (`is_key=true`, `position`, `source`). Deblochează filtru „cu niacinamidă" pe tabel (nu pe text).
- **`product_badges`**: din `attributes.badges` (vegan/cruelty-free/dermatologic testat) — fiecare cu proveniență (din `claim_provenance` kind=badge). *NB: badges nu-s încă populate în 168e-1 → autoring aici + provenance.*
- **`reviews`** individuale: 2-4/produs, derivate/autorate din `reviewSummary` (top_pros/top_cons → recenzii plauzibile) cu `source='seed_demo'`, autor demo, rating.
- **Seed:** extinde `seed_catalog_v2.py` (sau job separat) — idempotent, tranzacțional; sub gate-ul `evaluate(v3)`.
- **Consumatori (în bot):** `get_product_details` randează sections + INCI + badges + review-intel (leagă cu NX-169 proiecție).
