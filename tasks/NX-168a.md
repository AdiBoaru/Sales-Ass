# NX-168a — Catalog v2: schema atribute canonice + audit-gate STATIC

**Owner:** S · **Faza:** P1 · **Track:** Demo Data Overhaul · **Branch:** `feat/NX-168a-catalog-audit` · **Complexitate:** M · **Estimare:** 3-4h

## Goal
Primul livrabil independent al lui NX-168 (vezi `tasks/NX-168.md`): contractul de date v2 (`catalog_v2.schema.json`) + un **audit-gate static** (`scripts/audit_catalog_v2.py`) care rulează pe FIȘIERUL JSON (zero DB, zero retrieval) și **pică seed-ul** la incoerență. Blochează recăderea catalogului demo în incoerența care a produs bug-ul „makeup → păr". NU depinde de seed sau de NX-167 (partea LIVE a auditului — „retrieval real nu întoarce păr" — vine în 168b).

## Business Context
Catalogul demo templatat (500 produse, nume-colaj, atribute care nu se leagă de filtre) face botul să pară incompetent în demo. Auditul static e plasa care garantează că v2 (168b) și orice re-seed viitor respectă contractul: atribute canonice, nume curate, coerență categorie. Rulabil în CI/pre-commit → regresie prinsă înainte de deploy.

## Technical Description
**Contract (`db/seed/catalog_v2.schema.json`, JSON Schema draft 2020-12):** structură (brands/categories/products) + enum-uri CANONICE aliniate cu engine-ul (`src/domain/defaults/beauty_salon.json`): `concerns ∈ concern_map` (`dry`/`oily`/…, NU text RO), `finish ∈ {natural,matte,dewy,satin}`, `coverage ∈ {light,medium,full,buildable}`, nume fără sufix numeric.

**Audit static (`scripts/audit_catalog_v2.py`, Python pur — fără `jsonschema` obligatoriu):** 6 reguli determinist-derivate din date + contract (fără wordlist NLP), fiecare o funcție pură testabilă:
- **R1** concerns/finish/coverage în afara enum-ului canonic (bug-ul „ten uscat" vs `dry`).
- **R2** atribute-cheie lipsă per categorie (fond fără `finish`/`coverage`; skincare fără `concerns`).
- **R3** nume generate/duplicate (sufix numeric `\d{2,4}$` / nume identic normalizat).
- **R4** coerență nume↔categorie (familia de tip din nume ≠ familia categoriei — „Pensula" în „Fond de ten").
- **R5** `categorySlugs` în ramuri diferite de primary (root-branch din arborele de categorii — `par` pe un produs de `machiaj`).
- **R6** pereche din aceeași categorie fără diferențiator real (preț/rating/pros/cons identice).

`audit(data) -> {rule: [violări]}` pur; `main()` întoarce exit 1 dacă total>0 (gate). Validare de schemă opțională (soft-import `jsonschema` dacă e prezent).

**Dovadă (rulat):** pe `db/seed/catalog.json` legacy → **695 violări, exit 1** (R3=578, R6=61, R5=18, R4=13, R2=5). Pe `db/seed/catalog_v2.sample.json` (fixture minimal valid) → **0 violări, exit 0**.
