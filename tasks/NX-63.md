# NX-63 — Onboarding canal Telegram demo + rulare de test pe VPS
**Owner:** S + Adi (manual BotFather) · **Faza:** MVP (test multi-canal) · **Ord:** după NX-61 + NX-62 · **Branch:** chore/NX-63-telegram-onboarding · **Complexitate:** S · **Estimare:** 3h

## Goal
Un bot Telegram real, legat de business-ul demo, rulând pe VPS — ca să testăm
conversația cap-coadă vorbind direct cu botul (inbound → pipeline → echo/agent → reply).

## Business Context
Prima buclă de feedback pe produs cu un om real (Adi) care vorbește cu botul, fără
costurile/birocrația WhatsApp. Validează tot lanțul pe care l-am construit
(ingestie → worker → pipeline → outbox → dispatcher) pe un canal live.

## Technical Description
Demo are nevoie de un rând `channels` pentru Telegram (analog gap-ului WhatsApp
`channels=0`): `kind='telegram'`, `provider_account_id=<bot id>`, `business_id=demo`,
`status='active'`. Tokenul botului vine din `TELEGRAM_BOT_TOKEN` (config; pt demo
single-bot e suficient un token global — la multi-tenant, per canal în secret manager).

`scripts/seed_telegram_channel.py`: idempotent (ON CONFLICT pe `(kind,
provider_account_id)`) — inserează canalul demo. Bot id-ul se obține din
`getMe` cu tokenul. Rulează cu rol admin (DDL/seed, nu bot_runtime).

Pe VPS: `docker compose up` pornește `telegram-poller` (NX-61) + `worker` +
`dispatcher` + `redis`. Pollerul ia mesajele, pipeline-ul răspunde echo (până
intră G3 cu agentul real). Postgres rămâne Supabase remote.

## Principii CLAUDE.md aplicabile
P7 (token în `.env`/secret manager, niciodată în repo) · P12 (chat.id intră în
`channel_identities`, nu în loguri).

## Implementation Steps
1. **Manual (Adi):** BotFather → `/newbot` → nume + username → copiază tokenul în
   `.env` pe VPS (`TELEGRAM_BOT_TOKEN`). (Long polling → fără setWebhook/HTTPS.)
2. `scripts/seed_telegram_channel.py`: `getMe` → bot id; insert canal demo idempotent.
3. Rulează seed-ul o dată (local sau pe VPS) → confirmă rândul în `channels`.
4. Pe VPS: `docker compose up -d` (redis + worker + dispatcher + telegram-poller).
5. Test manual: trimite „salut" botului → primești echo-ul. Verifică `messages`
   (1 inbound + 1 outbound) și `analytics_events` pentru conversație.
6. Notează în README/runbook pașii de rulare a testului Telegram pe VPS.
