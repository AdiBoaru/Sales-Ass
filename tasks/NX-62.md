# NX-62 — Telegram outbound (TelegramClient) + rutare dispatcher pe canal
**Owner:** S · **Faza:** MVP (test multi-canal) · **Ord:** după NX-60 (paralel cu NX-61) · **Branch:** feat/NX-62-telegram-outbound · **Complexitate:** S · **Estimare:** 3h

## Goal
Dispatcher-ul trimite răspunsuri și pe Telegram, alegând clientul potrivit după
`channel_kind` — fără să dubleze logica de coadă/retry (aceeași pentru toate canalele).

## Business Context
Închide bucla de test pe Telegram: ce scrie pipeline-ul în outbox ajunge înapoi
în chat. Refolosește integral mașinăria dispatcher-ului (claim, backoff, visibility
timeout, dead-letter) — doar transportul diferă.

## Technical Description
După NX-60, `dispatch_row` alege sender-ul din `SENDER_REGISTRY[channel_kind]` și
cheamă `send_text(account_id, to, text)`. Aici adăugăm implementarea Telegram:

`src/channels/telegram/client.py` (extinde clientul din NX-61): `send_text(account_id,
to, text) -> str` apelează `POST /bot{token}/sendMessage` cu `chat_id=to`,
`text=text`; întoarce `str(result.message_id)`. `account_id` (bot id) e informativ —
tokenul vine din config (pt demo single-bot); la multi-tenant, tokenul per canal
din secret manager (ca la WhatsApp `credentials_ref`).

Înregistrare: `SENDER_REGISTRY['telegram'] = TelegramClient(...)` la bootstrap-ul
dispatcher-ului. `claim_due` întoarce deja `channel_kind` + `channel_account_id`
(NX-60), iar `payload['to']` = `chat_id` (pus de Sender din `sender_external_id`).
`mark_sent` + `set_message_provider_id` (wamid Telegram = message_id) rămân identice.

## Principii CLAUDE.md aplicabile
P5 (un singur punct de ieșire — outbox — cu transport pluggabil) · P6 (eșec de
trimitere → mark_failed cu backoff, nu pierdere tăcută; identic cross-canal).

## Implementation Steps
1. `TelegramClient.send_text(account_id, to, text)` → sendMessage → message_id.
2. Conformă cu `ChannelSender` Protocol (NX-60).
3. Bootstrap dispatcher: înregistrează clientul Telegram în registry (dacă
   `TELEGRAM_BOT_TOKEN` setat).
4. Verifică `dispatch_row` rămâne agnostic (alege pe `channel_kind`).
5. Test: TelegramClient.send_text cu httpx MockTransport → message_id; dispatch_row
   cu un canal telegram fake → folosește sender-ul telegram, marchează sent.
