# NX-95 — Detecție de limbă: bibliotecă în requirements (nu fallback regex)
**Owner:** S · **Faza:** P1 · **Zi/Ord:** după G5c (language_stage) · **Branch:** `feat/NX-95-langdetect-lib` · **Complexitate:** S · **Estimare:** 2.5h

## Goal
`detect_language` (stagiul 3) clasifică acum RO/HU/EN pe euristică de stopwords +
diacritice. E precision-first dar fragilă: mesaje scurte, fără stopwords din lista
mea sau cu mix de limbi cad pe `None` → păstrăm limba veche, care poate fi greșită.
Audit-ul (docs/ARCHITECTURE-AUDIT.md:66) a notat că o bibliotecă de detecție „model-grade"
NU e în `requirements.txt` → calea rulează DOAR pe euristică. Taskul aduce un detector
determinist real (bibliotecă) ca strat PRIMAR, cu euristica actuală păstrată ca PLASĂ
de siguranță. Zero LLM (principiul 2 — detector statistic determinist, nu generare).

## Business Context
Limba e parte din cheia de cache, faqs și triaj (principiul 11). Un mesaj clasificat
greșit ca RO când userul scrie HU înseamnă cache hit în limba greșită, FAQ în limba
greșită și un răspuns care sună străin → conversație pierdută. Verticalul beauty din
Transilvania are trafic HU real amestecat cu RO. Un detector solid pe primele 1-2
fraze e diferența între „botul vorbește limba mea" și „botul nu mă înțelege". Cost
zero la runtime (offline, deterministic) → nu mișcă deloc factura LLM.

## Technical Description

### Starea curentă (REAL, `src/lang/detect.py`)
`detect_language(text, supported) -> str | None`: tokenizează lowercase, intersectează
cu `_STOPWORDS[lang]`, adaugă `_DIACRITIC_BONUS=2` dacă apar diacritice specifice,
întoarce limba cu scor maxim DOAR dacă e `≥1` și strict peste a doua (tie → `None`).
`language_stage` (src/worker/stages/language.py) apelează `detect_language(ctx.message.body,
ctx.business.supported_locales)`; pe `None` SAU pe „aceeași limbă" face no-op (păstrează
`ctx.language`), altfel setează `ctx.language`, emite `language_detected` și persistă pe
conversație (best-effort). Cod pur, ZERO I/O, nu aruncă pe input.

### Ținta — detector bibliotecă PRIMAR + euristica ca plasă
Aleg **`lingua-language-detector`** (pip: `lingua-language-detector`): pur-Python,
fără model binar de descărcat la runtime (spre deosebire de fasttext, care cere `.bin`),
deterministic, precizie bună pe fraze scurte și suport explicit RO/HU/EN. Funcționează
offline, fără rețea, fără cheie. Construim un detector restrâns la limbile suportate
ale tenantului (lingua scade dramatic confuziile când lista de candidați e mică).

Rescriem `detect.py` astfel încât biblioteca să fie stratul PRIMAR și euristica de azi
să rămână funcția de plasă (redenumită intern, NU ștearsă):

```python
# src/lang/detect.py  (țintă)
from __future__ import annotations
import logging
log = logging.getLogger(__name__)

try:
    from lingua import IsoCode639_1, Language, LanguageDetectorBuilder
    _LINGUA_OK = True
except ImportError:                       # lib absentă (build minimal) → doar plasa
    _LINGUA_OK = False

_LANG_BY_CODE = {"ro": "ROMANIAN", "hu": "HUNGARIAN", "en": "ENGLISH"}  # mapăm intern
_DETECTOR_CACHE: dict[tuple[str, ...], object] = {}   # per combinație de supported

# încredere minimă: sub prag → None (precision-first, ca azi). Calibrat empiric.
_MIN_CONFIDENCE = 0.65
_MIN_CHARS = 3                            # sub 3 caractere "reale" → semnal prea slab

def _heuristic_detect(text, supported):   # = funcția de AZI, mutată 1:1 (plasa)
    ...                                   # stopwords + diacritice, neschimbată

def _lib_detect(text: str, supported: list[str]) -> str | None:
    langs = tuple(c for c in ("ro", "hu", "en") if c in supported)
    if len(langs) < 2:
        return None                       # mono-lingv tratat în stage; aici e defensiv
    det = _DETECTOR_CACHE.get(langs)
    if det is None:
        wanted = [getattr(Language, _LANG_BY_CODE[c]) for c in langs]
        det = LanguageDetectorBuilder.from_languages(*wanted).build()
        _DETECTOR_CACHE[langs] = det      # build e scump → cache per proces
    confidences = det.compute_language_confidence_values(text)
    if not confidences:
        return None
    top = confidences[0]
    if top.value < _MIN_CONFIDENCE:
        return None                       # incertitudine → None (NU ghicim)
    code = top.language.iso_code_639_1.name.lower()   # 'RO' -> 'ro'
    return code if code in supported else None

def detect_language(text: str | None, supported: list[str]) -> str | None:
    """Determinist, fără LLM. Strat 1 = bibliotecă (lingua); strat 2 = euristica
    veche ca plasă (lib absentă / sub prag de încredere / text foarte scurt)."""
    if not text or len(text.strip()) < _MIN_CHARS:
        return None
    if _LINGUA_OK:
        try:
            hit = _lib_detect(text, supported)
        except Exception:                 # noqa: BLE001 — detectorul NU rupe turul
            log.warning("langdetect: lib a eșuat → fallback euristic")
            hit = None
        if hit is not None:
            return hit
    return _heuristic_detect(text, supported)   # plasa: contract identic (str|None)
```

Contractul rămâne IDENTIC: `str | None`, cod pur, nu aruncă, precision-first. `language_stage`
NU se atinge — toată logica de no-op/persist/emit e deja corectă acolo. Singura schimbare
de comportament e că `detect_language` întoarce un rezultat bun pe MAI multe mesaje (mai
puține `None` nejustificate), iar plasa euristică prinde exact cazurile de azi când lib e
sub prag.

### Limită de încredere = anti-ghicit
`_MIN_CONFIDENCE=0.65` (configurabil prin settings, vezi mai jos) ține contractul
precision-first: lingua întoarce încredere normalizată per candidat; sub prag → `None`,
deci `language_stage` păstrează limba curentă. Pe text foarte scurt (`< _MIN_CHARS`) sărim
complet — un „ok" de 2 litere nu schimbă limba conversației.

### Config (kill-switch, ca restul stagiilor)
`langdetect_lib_enabled` (default `True`) — pe `False` rulează DOAR euristica (escape
hatch dacă lib se comportă prost în prod, fără redeploy de cod). `langdetect_min_confidence`
(default `0.65`). `_lib_detect` citește pragul din settings; `detect.py` rămâne import-safe
(nu importă settings la nivel de modul — îl citește în funcție, ca să nu lege un modul de
cod pur de mediul de rulare).

## Test Cases
> Toate deterministe, offline. Lingua NU face apeluri de rețea (model încorporat în pachet)
> → e sigur de rulat în CI fără mock pe rezultate. Cazurile de eroare folosesc monkeypatch
> pe `_lib_detect`. ZERO apeluri LLM/rețea în CI.

**Happy Path:**
1. `detect_language("Szia, mennyibe kerül?", ["ro","hu","en"]) == "hu"` (lib, încredere mare).
2. `detect_language("do you ship to Cluj", ["ro","hu","en"]) == "en"` — fără stopword EN
   din lista veche în „ship/Cluj", dar lib clasifică corect (caz care AZI dădea `None`).

**Edge Cases:**
1. `langdetect_lib_enabled=False` (settings monkeypatch) → folosește euristica; RO clasic
   („caut o cremă de față") tot `"ro"`.
2. Text ambiguu scurt („ok 👍") → `None` (sub `_MIN_CHARS` / sub prag), limba conversației
   nu se schimbă în `language_stage`.
3. Tenant mono-lingv `supported=["ro"]` cu input HU → `None` (nu inventăm limbă neacceptată).

**Failure Cases:**
1. `_lib_detect` aruncă (monkeypatch RuntimeError) → `detect_language` prinde, cade pe
   `_heuristic_detect`, întoarce rezultatul euristic SAU `None`, fără excepție propagată.
2. Lib întoarce o limbă în afara `supported` (ex. detector larg, edge) → mapare prinde
   `code not in supported` → `None`, NU o limbă neacceptată.
3. `_LINGUA_OK=False` simulat (import eșuat) → `detect_language` rulează doar euristica;
   pipeline-ul funcționează identic cu starea pre-NX-95.

## Cost
**$0 / 1000 mesaje.** Detectorul lingua e offline și determinist — niciun token LLM,
niciun apel de rețea, nicio cheie. Ce limitează „costul" e CPU: build-ul detectorului
e cache-uit per combinație `supported` (`_DETECTOR_CACHE`), deci se plătește o singură
dată per proces, nu per mesaj; clasificarea per mesaj e sub-milisecundă pe fraze scurte.
Nu mișcă `daily_cost_cap_usd` și nu apare în `usage_daily.cost_usd`.
