  (rate-limit/cost-guard rulează cu logica reală, fără infra). 9 scenarii × verificări de contract.

## Rezultatele auditului (calea /web/chat reală)

**Merg corect pe web (verificat, nu presupus):**
- **safety NX-173** ține pe web: 0 carduri cu retinoizi pe „sunt însărcinată", trimitere la medic
  exact o dată (era verificat DOAR prin sim/whatsapp înainte — acum și pe canalul real);
- **NX-178 diacritice** e **mascat de retrieverul VECTOR**: „sampon" → 3 carduri de șampon pe web
  (leg-ul lexical dă 0, dar hibridul salvează) → **P2, nu P0**. Contează pt exactitatea lexicală,
  dar clientul primește rezultate;
- link → `offer(open_url)` cu URL real; order-anon → zidul de login; nonsense → clarificare cu chips
  (P6 respectat peste tot: 0 răspunsuri goale).

**Defecte confirmate pe web (carduri separate):**
- **NX-175** (FAQ retur): „Cum pot face un retur?" → răspunde **«Nu.»** (alege excepția „desigilat")
  + typo „14 zile. calendaristice." ajunge la client. Cel mai vizibil.
- **NX-174** (compare): acum afișează **ambele** carduri (progres!), dar **dublează** numele
  produselor în `content` — contractul iZi cere `content` = doar framing, enumerarea = în carduri.
- **NX-176** (rutină): „fă-mi o rutină de machiaj" → „nu am exact ce cauți" + pensule/spray, nicio
  rutină reală.
