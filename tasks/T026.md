# T026 — 002: contacts last_inbound_at / consent / erased_at
**Owner:** S · **Faza:** MVP · **Zi:** Z1-2 · **Branch:** `feat/T026-sql-contacts` · **Complexitate:** XS · **Estimare:** 0.5h

## Goal
Trei coloane mici cu trei roluri mari: last_inbound_at alimentează fereastra de 24h Meta (fără ea proactivul trimite orbește și mesajele pică), consent ține opt-in-ul pentru mesaje proactive (obligație legală, nu feature), erased_at marchează contactele anonimizate GDPR.

## Business Context
24h window = regula de fier a platformei WhatsApp; consent = GDPR/ePrivacy; ambele sunt condiții de a opera, nu opțiuni.

## Technical Description
```sql
alter table conv.contacts
  add column last_inbound_at timestamptz,
  add column consent jsonb not null default '{}'::jsonb,
  add column erased_at timestamptz;
-- backfill last_inbound_at din istoricul existent:
update conv.contacts c set last_inbound_at = sub.max_at
from (select conversation_id, max(created_at) max_at
      from conv.messages where role = 'user' group by 1) sub
where sub.conversation_id = c.conversation_id;
```
Formatul consent: `{"proactive": {"granted": true, "at": "...", "source": "first_message_optin"}, "marketing": {...}}` — granular pe tip, cu timestamp și sursă (dovadă la audit GDPR). Funcția helper (în cod, T201): `in_24h_window = last_inbound_at > now() - interval '24 hours'` — DERIVAT, niciodată stocat ca boolean (un flag stocat minte la graniță; un timestamp nu minte niciodată).
