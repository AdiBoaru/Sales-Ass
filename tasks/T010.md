# T010 — Dockerfile multi-stage
**Owner:** S · **Faza:** MVP · **Zi:** Z1 · **Branch:** `chore/T010-dockerfile` · **Complexitate:** S · **Estimare:** 1.5h

## Goal
O singură imagine pentru webhook și worker (comanda diferă), mică, non-root, cu layer caching pe requirements ca rebuild-urile să fie secunde, nu minute.

## Business Context
Aceeași imagine merge în dev, staging și prod — ce testezi e ce livrezi.

## Technical Description
Stage 1 `builder`: python:3.12-slim, `pip install --prefix=/install -r requirements.txt`. Stage 2 `runtime`: python:3.12-slim, copy /install → /usr/local, copy src/, user non-root `app` (uid 1000), `WORKDIR /app`, fără CMD hardcodat pe webhook (comanda vine din compose). `.dockerignore`: .venv, .git, tests, __pycache__, .env*.

## Principii CLAUDE.md aplicabile
Un singur artefact de build pentru ambele procese — reduce suprafața de „diferențe între medii".

## Implementation Steps
1. Scrie Dockerfile + .dockerignore.
2. `docker build -t nativx:dev .` → verifică dimensiunea (<300MB).
3. Verifică user non-root: `docker run --rm nativx:dev id` → uid 1000.
4. PR + merge.
