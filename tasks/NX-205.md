# NX-205 — Contract Facts / Evidence / Provenance / DerivedSignals

**Owner:** Claude (build) / Codex (verify) · **Faza:** 2 (Quality Overhaul) · **Status:** DRAFT — nu se implementează până la review dependențe + DoD · **Complexitate:** M
**Parent plan:** [docs/QUALITY-OVERHAUL-2026.md](../docs/QUALITY-OVERHAUL-2026.md) · **Depinde de:** NX-200 (ADR aprobat), **NX-168d (REUSED — contractul v3 + `claim_provenance` + `not_recommended_for{level,source,verified_at}` EXISTĂ deja acolo)**
**Decizii ADR aplicabile:** D3, D4, D5, D9
**⚠️ NU rescrie NX-168d.** Acest card **extinde** contractul existent cu: `evidence_chunks` (roluri),
`DerivedSignals{rule_id}`, `schema_version`/`locale` pe artefacte derivate. Vocabularul canonic
(concerns, finish, coverage, texture, routine_step) rămâne al 168d, sursă unică.

## Goal
Contractul tipizat al adevărului despre produse — fundația pe care se construiesc F3 (catalog),
F4 (documente derivate) și F8 (validare claims).

## Scope
1. **Facts canonice per categorie** (din DomainPack, nu hardcodat): finish, texture, coverage,
   suitable_for, concerns, free_of, key_ingredients, usage, wear_time, net_content,
   not_recommended_for, variants (+ preț/stoc = facts live, nu snapshot).
2. **Provenance pe claims importante:** `{facet, value, source, source_ref, verified_at}` —
   obligatoriu pentru safety/free_of/contraindicații (D5).
3. **DerivedSignals separate de fapte confirmate:** `{signal, derived_from[], rule_id}` —
   regulile de derivare versionate, reparabile global.
4. **Evidence chunk contract:** `{role: benefit|usage|warning|ingredient|faq|review_summary|policy,
   text, source, product_id}` — negativele INTRĂ aici (D9).
5. Toate contractele poartă `business_id`, `locale`, `schema_version` (D3). Validatoare Pydantic.

## Out of Scope
Popularea datelor (NX-206) și generarea documentelor (NX-207).

## Definition of Done
- [ ] Contracte Pydantic + migrare SQL (ADITIVĂ) aprobate de Codex; delta față de NX-168d explicit.
- [ ] Câmpuri obligatorii per categorie definite în DomainPack (nu hardcodat).
- [ ] Validatoare de contract cu teste (inclusiv respingerea contradicțiilor + claims fără sursă).
- [ ] Zero breaking change pe schema existentă; auditul v2/v3 din 168d rămâne verde.
- [ ] Invariante: `business_id` pe toate tabelele/queries + test izolare; zero PII în contracte;
      `locale` + `schema_version` prezente pe artefacte; `ruff` + `ruff format --check` + `pytest` verzi.
