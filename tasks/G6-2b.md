# G6-2b — Context builder bogat (felia 2): summarizer conversații lungi
**Owner:** S · **Faza:** P1 · **Branch:** `feat/G6-2b-summarizer` · **Complexitate:** M · **Estimare:** ~5h

## Goal
Felia 2 a stagiului 6: pentru conversațiile lungi (>20 mesaje), un **rezumat rolling** comprimă
mesajele care ies din fereastra de ultimele 8 → triaj/agent țin firul FĂRĂ să trimită zeci de
mesaje vechi (CLAUDE.md: „>20 mesaje → `conversation_summaries` + ultimele 8"). Generarea rulează
**post-tur async** (nano), citirea e deterministă în context builder.

## Business Context
Pe conversații lungi de vânzare, fără rezumat botul „uită" începutul (doar ultimele 8 ajung la
model). Rezumatul rolling păstrează contextul (ce caută, ce s-a discutat, constrângeri) la cost
mic și reduce tokenii pe conversațiile lungi.

## Technical Description (design verificat adversarial — 3 bug-uri reparate față de planul inițial)

### Generare — post-tur async (`processor._summarize_if_needed`)
- Rulează DUPĂ outbox + `_cache_writeback` (nu întârzie livrarea). NANO (`llm.complete(model=
  model_triage)`) — în afara pipeline-ului sincron, deci P2 (LLM doar triaj+agent) rămâne respectat;
  consistent cu „POST-TUR async extractor profil nano" (stagiul 9). best-effort try/except.
- Skip dacă `summary_enabled=False` sau `llm is None` (cost guard tripped / fără cheie) → P6+P2 gratis.
- **FIX#1 (acoperire):** sumarizează `get_messages_for_summary(after=watermark, tail=8)` = mesajele
  care ies din ultimele 8 — NU `ctx.history` (capat la 8, ar suma aceleași 8 din transcript).
- **FIX#2 (watermark onest):** `upto_message_at = to_summarize[-1].created_at` = cel mai NOU mesaj
  INCLUS, nu `MAX(created_at)` global (altfel ar pretinde că acoperă mesaje nevăzute).
- **FIX#3 (cost guard):** după insert reușit → `cost_add(redis, biz, cost_triage_usd)` (G2c) +
  `insert_events('summarizer_run')` — apelul nano nu scapă contorului zilnic.
- Anti-regenerare: dacă există rezumat anterior și mesajele noi < `summary_regen_delta` → skip
  (limbo temporar bounded acceptat; la regenerare feed-ul ia TOT de la watermark → fără pierderi).

### Generare — logica izolată (`src/worker/summarizer.py`)
- `build_summary_prompt` (system static + user: rezumat anterior + mesaje noi), `generate_summary`
  (NANO, întoarce None la gol). Redactare PII (`_PHONE_RE`) pe input ȘI output + instrucțiune
  anti-telefon în system (P12 defense-in-depth).

### Citire — deterministă (`context.summary_block` + processor seed)
- Processor seedează `ctx.summary` (un singur SELECT indexat/tur, `get_summary_for_context`).
- `summary_block(ctx)` (I/O-free) formatează din `ctx.summary`, bugetat la `summary_max_chars` (600).
- `context_blocks` pune rezumatul PRIMUL (cronologic: fundal vechi → profil → state → transcript
  downstream). În USER (dinamic) → SYSTEM byte-identic, prompt caching neatins.

### Query-uri
- `messages.count_messages`, `messages.get_messages_for_summary` (window `row_number() desc`, rn>tail
  + created_at>after). `summaries.get_latest_summary` / `get_summary_for_context` /
  `insert_conversation_summary` (append-only checkpoints). Toate `business_id=$1` (P7).

### Config
`summary_enabled` (True), `summary_threshold` (20), `summary_regen_delta` (12), `summary_max_chars` (600).
