## Goal
Abonamentele „anunță-mă când revine pe stoc" — tabelul pe care tool-ul (T122) scrie și proactivul (T204) citește. Schema acum, logica în P1.

## Business Context
Fiecare „nu mai avem pe stoc" fără captură de intenție = client pierdut definitiv. Cu abonament, devine vânzare amânată — una dintre cele mai profitabile bucle ale botului.

## Technical Description
```sql
create table conv.back_in_stock_subscriptions (
  id uuid primary key default gen_random_uuid(),
  business_id uuid not null references core.businesses(id),
  contact_id uuid not null references conv.contacts(id) on delete cascade,
  product_id uuid not null,
  variant_id uuid,
  notified_at timestamptz,                 -- NULL = încă așteaptă
  created_at timestamptz not null default now(),
  unique (business_id, contact_id, product_id, variant_id)
);
create index idx_bis_pending on conv.back_in_stock_subscriptions(product_id)
  where notified_at is null;
```
NULLS NOT DISTINCT atenție: pe Postgres 15+, unique cu variant_id NULL permite duplicate — folosește `unique nulls not distinct (...)` ca dublu-subscribe fără variantă să fie idempotent.
