# NX-74 — Strat gratuit FAQ + tool faq_lookup (faqs)
**Owner:** S · **Faza:** P1 · **Zi/Ord:** după G5b (cache) + G7 (tool-uri) · **Branch:** `feat/NX-74-faq-layer` · **Complexitate:** M · **Estimare:** 4h

## Goal
Întrebările de cunoștințe („care e politica de retur?", „cât costă livrarea?", „aveți factură?")
n-au azi nicio cale FAQ — cad pe triaj+agent (cost) sau pe fallback. Acest task adaugă tabelul
`faqs` în pipeline pe DOUĂ căi: (1) ca **strat gratuit** (stagiul 4, ÎNAINTE de triaj/agent →
early-exit fără LLM de generare) și (2) ca **tool al agentului** (`faq_lookup(query)`, când
agentul e deja pe o rută de vânzare și are nevoie de un fapt de business). Lookup-ul e
ÎNTOTDEAUNA `business_id + locale + cosine pe embedding(1536)`, embedding-ul calculat prin
adaptorul unic. `faqs=0` pe demo, dar codul e necesar oricum — fără el nu există cale FAQ
indiferent câte rânduri seedăm.

## Business Context
40-60% din trafic trebuie să se oprească în straturile gratuite (CLAUDE.md, stagiul 4). FAQ-ul
e exact tipul de întrebare repetată, ieftin de servit dintr-o bază curată, fără să ardem
triaj(nano)+agent(mini) pe fiecare „cât e livrarea". Pentru retaileri (beauty/HVAC/auto)
politica de retur, livrare, garanție și facturare sunt 90% din întrebările non-produs — un
strat FAQ bine pus scade costul per conversație și ține răspunsurile consistente (un singur
adevăr editat de client în dashboard, nu halucinat de model).

## Technical Description

### Stare curentă vs țintă
- **Curent:** NU există `src/db/queries/faqs.py`, NU există tool `faq_lookup`, NU există stagiu
  FAQ. Tabelul `faqs` există în schema_v2 (liniile 488-501), cu `embedding vector(1536)`,
  `locale text not null default 'ro'`, `is_active boolean`, index HNSW cosine `idx_faqs_emb`.
- **Țintă:** un query nou + un stagiu nou (free layer) + un tool nou + schema OpenAI a tool-ului.

### 1. Query layer — `src/db/queries/faqs.py` (NOU)
Tiparul e identic cu `semantic_cache.semantic_lookup` (cosine + `business_id` + `locale`),
refolosim convenția `_vec()` (literal pgvector `[a,b,c]` cu 7 zecimale). Stratul de date NU
apelează LLM — primește vectorul gata calculat (principiul 2). `conn` e DEJA tenant-scoped.

```python
def _vec(embedding: list[float]) -> str:
    return "[" + ",".join(f"{x:.7f}" for x in embedding) + "]"

async def semantic_lookup(
    conn, business_id: str, locale: str, embedding: list[float], *, limit: int = 1
) -> dict[str, Any] | None:
    """Cel mai apropiat FAQ activ (cosine) pe (business_id, locale). None la 0 rânduri.
    Caller-ul aplică pragul de similaritate."""
    row = await conn.fetchrow(
        """
        select id::text as id, question, answer,
               1 - (embedding <=> $3::vector) as similarity
        from faqs
        where business_id = $1
          and locale = $2
          and is_active = true
          and embedding is not null
        order by embedding <=> $3::vector
        limit 1
        """,
        business_id, locale, _vec(embedding),
    )
    return dict(row) if row else None
```
`embedding is not null` în WHERE evită rândurile ne-embed-uite (seed parțial); ele nu pot fi
ordonate cosine oricum. NU expunem `business_id` ca parametru opțional — e mereu `$1`. RLS pe
`bot_runtime` (003) e plasa: un query fără filtru → 0 rânduri, nu datele altui client.

### 2. Strat gratuit — `src/worker/stages/faq.py` (NOU stagiu)
Stagiu separat de `cache.py` (separarea proprietarilor): `cache_stage` deține `semantic_cache`,
`faq_stage` deține `faqs`. Plasare în pipeline DUPĂ `cache_stage` și ÎNAINTE de `triage_stage`.
Ordinea: cache (cel mai ieftin, O(1) pe exact) → FAQ (un embed) → triaj. Best-effort, ca
cache-ul: orice eroare (migrare neaplicată, DB, embed) → degradare la „miss", NU rupe turul
(principiul 6).

```python
async def faq_stage(ctx: TurnContext, deps: PipelineDeps) -> None:
    s = get_settings()
    if not s.faq_enabled:
        return
    body = (ctx.message.body or "").strip()
    if not body or deps.llm is None:        # fără LLM nu putem embed → miss grațios
        return
    try:
        emb = (await deps.llm.embed([body]))[0]
        hit = await semantic_lookup(deps.conn, ctx.business.id, ctx.language, emb)
        sim = float(hit["similarity"]) if hit else 0.0
        if hit is not None and sim >= s.faq_tau_high:
            ctx.set_reply(hit["answer"])     # cacheabil (răspuns static reutilizabil)
            ctx.emit("faq_hit", faq_id=hit["id"], similarity=round(sim, 4))
            return
        ctx.emit("faq_lookup", layer="miss", similarity=round(sim, 4))
    except Exception as e:  # noqa: BLE001 — FAQ best-effort
        log.warning("faq: lookup eșuat (%s) → miss", type(e).__name__)
```
Pragul `faq_tau_high` e separat de `cache_tau_high` (FAQ-ul e curatat manual de client → poate
fi puțin mai relaxat, dar îl ținem precision-first; default 0.82, tunabil în config). Doar
`embed()`, niciodată generare (principiul 2). `set_reply` cu `cacheable=True` implicit: un
hit FAQ e exact tipul de răspuns static reutilizabil pe care G5b vrea să-l prindă data viitoare.

**Câmpuri TurnContext scrise:** `ctx.reply` (via `set_reply`, la hit) + `ctx.events` (via
`emit`). NU scrie `ctx.route`, `ctx.retrieval`, `ctx.from_cache` — nu e proprietarul lor.

### 3. Tool al agentului — `src/tools/faq_tools.py` (NOU) + schema în `tool_definitions.py`
Pe ruta SALES agentul poate avea nevoie de un fapt de business („pot plăti ramburs?") în
mijlocul unei recomandări. `faq_lookup` refolosește ACELAȘI query (`faqs.semantic_lookup`),
contract uniform `async def tool(ctx, deps, args) -> ToolResult`, `business_id`/`locale` din
`ctx` (modelul NU le primește — principiul 7). FAQ-ul nu produce produse → `products=[]`,
doar `llm_view` (textul faptului) pentru model.

```python
class FaqArgs(BaseModel):
    query: str = Field(min_length=1)

@register("faq_lookup")
async def faq_lookup_tool(ctx, deps, args) -> ToolResult:
    a = FaqArgs(**args)
    if deps.llm is None:
        return ToolResult(ok=False, error="no_llm", llm_view="FAQ indisponibil.")
    emb = (await deps.llm.embed([a.query]))[0]
    hit = await semantic_lookup(deps.conn, ctx.business.id, ctx.language, emb)
    if hit is None or float(hit["similarity"]) < get_settings().faq_tau_tool:
        return ToolResult(ok=True, llm_view="Nu am un răspuns în baza de cunoștințe; "
                          "spune-i clientului că verifici cu un coleg dacă insistă.")
    return ToolResult(ok=True, llm_view=hit["answer"])
```
Pragul de tool `faq_tau_tool` poate fi puțin mai jos decât pragul de strat gratuit (agentul
parafrazează oricum răspunsul în context, deci un match aproximativ e util; default 0.75).
Tool-ul se adaugă în `_SALES_TOOLS` (în `src/tools/base.py`), import în `agent.py` (linia
`from src.tools import ...` — importul înregistrează tool-ul), și o intrare în `_SCHEMAS`
(`tool_definitions.py`) cu `strict: True` și un singur câmp `query` (string). Descrierea
schemei: „Caută în baza de cunoștințe a magazinului un fapt de business (livrare, retur,
garanție, plată, facturare). Folosește când clientul întreabă o regulă/politică, NU pentru
produse." Mențiune scurtă în `_TOOL_SYSTEM` din `agent.py`.

### Error handling & redactare loguri
- Stratul gratuit: tot corpul try/except → miss la orice eroare (best-effort, ca G5b).
- Tool: `run_tool` (base.py) prinde deja excepțiile → `ToolResult(ok=False)`, nu rupe turul.
- Loguri: NICIODATĂ corpul mesajului sau întrebarea în loguri/evenimente — doar `faq_id` +
  `similarity` (principiul 12). FAQ-ul nu are PII (e text de business, nu de contact), dar
  ținem regula uniform.

## Principii CLAUDE.md aplicabile
- **P2 (LLM doar triaj+agent):** FAQ folosește DOAR `embed()` (clasificator/căutare, ca G5b
  și search semantic) prin adaptorul unic `src.agent.llm` — NU generare. Singurul loc care
  vorbește OpenAI rămâne adaptorul; stratul de date primește vectorul gata calculat.
- **P3 (un proprietar per câmp):** `faq_stage` scrie DOAR `ctx.reply` + `ctx.events`. Stagiu
  separat de `cache_stage` ca să nu existe doi proprietari pe aceeași logică de „strat gratuit".
- **P4 (straturi gratuite 40-60%):** scopul direct — opri întrebările de cunoștințe înainte
  de orice LLM de generare.
- **P6 (niciodată tăcere):** miss/eroare → pipeline-ul continuă spre triaj/agent/fallback, nu
  tăcere.
- **P7 (business_id pe tot):** `semantic_lookup` are `business_id = $1` explicit; RLS plasa.
- **P11 (limba e parte din cheie):** lookup-ul include `locale = $2` (din `ctx.language`). Un
  hit FAQ în limba greșită e un BUG — filtrăm pe locale în ambele căi (strat + tool).

**Edge Cases:**
1. `similarity=0.6` < prag → `faq_stage` nu setează reply, emite `faq_lookup`/miss, pipeline
   merge mai departe spre triaj.
2. Lookup pe `locale='en'` când singurul FAQ e `locale='ro'` → 0 rânduri → None (filtrul pe
   `$2` exclude rândul; nu hit în limba greșită).
3. Body gol/None → `faq_stage` nu apelează `embed`, trece mai departe.

1. `deps.llm.embed` aruncă (API/rețea) → `faq_stage` prinde, log warning, miss; fără reply de
   FAQ, pipeline continuă (test verifică nicio excepție propagată).

   `ToolResult(ok=False, error="no_llm")` fără să atingă DB.
3. `semantic_lookup` (DB) aruncă → în `faq_stage` miss grațios; în tool, `run_tool` prinde →
   `ToolResult(ok=False)`, turul agentului continuă.

## Cost (embeddings)
Stratul FAQ adaugă **1 apel `embed()` (1 text scurt)** pe mesajele care ajung la el (cele
neprinse de gates/cache). `text-embedding-3-small` = ~$0.00002 / 1K tokeni; un mesaj de chat
~20-40 tokeni → ~$0.0000008 / mesaj. La **1000 mesaje** care ating stratul: **~$0.001** (sub
un cent). Tool-ul `faq_lookup` adaugă încă un `embed()` DOAR când agentul îl cheamă (rar, doar
pe rute sales cu întrebare de business).

**Ce limitează costul:**
- Gates + `cache_stage` rulează ÎNAINTE → mesajele prinse acolo nu mai ating embed-ul FAQ.
- Un singur `embed()` per mesaj (un text), nu per FAQ — costul nu crește cu numărul de FAQ-uri.
- `faq_enabled` (kill-switch în config) + `deps.llm is None` (fără cheie) → zero apeluri.
- Hit-urile devin cacheabile (`set_reply(cacheable=True)`) → a doua oară pot fi servite de G5b
  fără niciun embed FAQ.
- ECONOMIE netă: fiecare hit FAQ evită triaj(nano)+agent(mini), mult mai scumpe decât un embed.
