# T005 — CI: pytest
**Owner:** S · **Faza:** MVP · **Zi:** Z1 · **Branch:** `chore/T005-ci-pytest` · **Complexitate:** XS · **Estimare:** 1h

## Goal
Al doilea status check: niciun PR fără teste verzi. De aici încolo, „merge în main" = „testele au trecut", fără excepții umane.

## Business Context
Botul răspunde clienților finali ai clienților voștri — o regresie nedetectată costă încrederea clientului plătitor.

## Technical Description
În `ci.yml`, job `test`: setup-python 3.12, cache pip pe hash(requirements.txt), `pip install -r requirements.txt`, `pytest -x -q`. Env-uri de test fake setate în workflow (`OPENAI_API_KEY=test-key` etc.) ca config-ul (T040) să booteze; testele NU ating servicii reale (fakeredis + LLM replay din T140; testele de integrare DB rulează doar local, marcate `@pytest.mark.integration` și excluse în CI cu `-m "not integration"`).

## Principii CLAUDE.md aplicabile
Regula 5 CONTRIBUTING. Decizia integration-only-local e temporară — se revizuiește în P1 (eventual DB de test efemer în CI).

## Implementation Steps
1. Adaugă jobul `test` în ci.yml cu env fake + `-m "not integration"`.
2. Adaugă un test trivial `tests/test_sanity.py` ca jobul să aibă ce rula.
3. PR, verifică rularea, merge.
4. Adaugă `test` la required checks.
