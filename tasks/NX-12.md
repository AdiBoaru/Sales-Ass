# NX-12 — Failover LLM: OpenAI → Anthropic în adapter (bibliotecă, nu serviciu)
**Owner:** S · **Faza:** P1 · **Ord:** O1 · **Branch:** feat/NX-12-llm-failover · **Complexitate:** L · **Estimare:** 16h

## Goal
Un incident OpenAI nu mai înseamnă „botul tace pentru toți clienții": agentul comută automat pe Anthropic și conversația continuă.

## Business Context
Principiul 6 („niciodată tăcere") aplicat la nivel de platformă, nu doar de retry. Diagrama cere „Model Gateway multi-provider" — implementarea corectă la scara actuală e o BIBLIOTECĂ în procesul workerului (zero hop de rețea, zero serviciu de operat), nu un microserviciu gateway.

## Technical Description
`src/llm/adapter.py` definește interfața unică: `async def complete(model_tier, messages, tools, response_format) -> LLMResult`. Provider 1: OpenAI (gpt-5.4-mini / nano). Provider 2: Anthropic (tier echivalent, configurat în settings). Traducerea schemelor: tools OpenAI (`function.parameters` JSON Schema) → tools Anthropic (`input_schema`), mesaje system/user/assistant/tool_result mapate în ambele direcții, `response_format json` → instrucțiune + extragere. Circuit breaker per provider: după N=5 eșecuri (timeout/5xx/429 repetat) într-o fereastră de 120s → OPEN 120s, traficul merge direct pe providerul 2; half-open cu o cerere de probă. Flag per business în `businesses.settings` (`llm_fallback_enabled`, default true) + kill-switch global în env. Costul/tokenii ambilor provideri intră identic în cost guard.

## Principii CLAUDE.md aplicabile
P6 (degradare controlată) · P3 (plafoane de cost se aplică indiferent de provider).

## Implementation Steps
1. Extrage interfața din clientul OpenAI existent (T101) în `adapter.py`; OpenAI devine `providers/openai.py`.
2. `providers/anthropic.py` cu maparea tools + messages; testează tool-calling real pe 3 tool-uri (search_products, get_product_details, request_human).
3. `breaker.py`: stare per provider în Redis (ca să fie partajată între workeri), nu în memorie.
4. Config: `LLM_PROVIDER_ORDER=openai,anthropic`, chei separate, tier mapping în settings.
5. Golden tests rulate o dată cu providerul forțat pe Anthropic → diferențele de ton documentate; promptul ajustat dacă un golden pică.
6. Metrice: `llm_calls{provider, outcome}` în analytics_events; alertă la breaker OPEN.

## Definition of Done

- [ ] Golden suite verde pe AMBII provideri

## Test Cases
**Happy:** OpenAI sănătos → zero apeluri Anthropic. **Edge:** eșec DOAR pe nano (triajul) → failover doar pe tier-ul afectat, mini rămâne pe OpenAI. **Failure:** ambii provideri down → scara de degradare existentă (template + handoff), eveniment `llm_all_providers_down`.
