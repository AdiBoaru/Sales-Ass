# T028 — 002: core.wa_templates
**Owner:** S · **Faza:** MVP · **Zi:** Z1-2 · **Branch:** `feat/T028-sql-templates` · **Complexitate:** XS · **Estimare:** 0.5h

## Goal
Ciclul de viață al template-urilor Meta (draft→submitted→approved→rejected/paused) trăiește în DB, per business, per limbă, versionat. Regula de platformă: în afara ferestrei de 24h, NUMAI template approved — deci proactivul (P1) are nevoie de sursa asta de adevăr.

## Business Context
Template respins de Meta descoperit în ziua trimiterii = campanie proactivă moartă. Lifecycle-ul în DB + sync-ul de status (T202) previne exact asta.

## Technical Description
```sql
create table core.wa_templates (
  id uuid primary key default gen_random_uuid(),
  business_id uuid not null references core.businesses(id),
  channel_instance_id uuid not null references core.channel_instances(id),
  name text not null,                       -- numele exact din Meta (snake_case)
  language text not null default 'ro',
  category text not null default 'utility'
    check (category in ('utility','marketing','authentication')),
  version int not null default 1,
  body text not null,                       -- cu placeholdere {{1}}, {{2}}
  variables jsonb not null default '[]',    -- descrierea fiecărui {{n}}
  provider_template_id text,
  status text not null default 'draft'
    check (status in ('draft','submitted','approved','rejected','paused','deprecated')),
  rejected_reason text,
  created_at timestamptz not null default now(),
  unique (business_id, channel_instance_id, name, language, version)
);
```
Categoria contează comercial: utility e ieftin și aprobat ușor (order updates), marketing e scump și capricios — separarea corectă la creare (T202) economisește bani reali.
