# T021 — 002: conv.outbox
**Owner:** S · **Faza:** MVP · **Zi:** Z1-2 · **Branch:** `feat/T021-sql-outbox` · **Complexitate:** S · **Estimare:** 1h

## Goal
Tabelul prin care iese ORICE mesaj din sistem. Sender-ul scrie aici tranzacțional (împreună cu state + message), dispatcherul citește și trimite. E fundația principiilor 5 (un singur punct de ieșire) și 6 (niciodată tăcere): dacă Meta sau Redis pică, mesajul există în DB și pleacă la retry.

## Business Context
Diferența dintre demo și producție: un mesaj „pierdut" către clientul final = ticket de suport la clientul plătitor = retainer în pericol.

## Technical Description
```sql
create table conv.outbox (
  id uuid primary key default gen_random_uuid(),
  business_id uuid not null references core.businesses(id),
  conversation_id uuid not null,
  idempotency_key text not null,
  kind text not null default 'message'
       check (kind in ('message','template','typing','reaction')),
  payload jsonb not null,            -- formatul EXACT trimis la Meta (text sau interactive)
  status text not null default 'pending'
       check (status in ('pending','dispatching','sent','failed','dead')),
  attempts int not null default 0,
  next_attempt_at timestamptz not null default now(),
  last_error text,
  sent_message_id uuid,              -- FK logic spre conv.messages după send
  created_at timestamptz not null default now(),
  unique (business_id, idempotency_key)
);
create index idx_outbox_due on conv.outbox(next_attempt_at)
  where status in ('pending','failed');
```
idempotency_key = `{turn_id}:{seq}` (un tur poate produce 2 mesaje la split). Dispatcherul face claim cu `UPDATE ... SET status='dispatching' WHERE id IN (SELECT ... FOR UPDATE SKIP LOCKED)` — SKIP LOCKED permite mai mulți dispatcheri fără double-send. Grant-uri: bot_runtime are SELECT/INSERT/UPDATE.
