# NX-02 — Redis durabil + securizat (AOF everysec, parolă, noeviction)
**Owner:** S · **Faza:** MVP · **Zi:** Z2 · **Branch:** chore/NX-02-redis-aof · **Complexitate:** XS · **Estimare:** 2h

## Goal
Un restart de Redis nu pierde niciun mesaj de client aflat în stream și nimeni neautentificat nu se poate conecta la backbone.

## Business Context
S4 din audit: azi Redis rulează cu config implicit — fără persistență (RDB rar sau deloc), fără parolă, eventual cu port publicat. Mesajele „în zbor" (primite, neprocesate) sunt date de producție: pierderea lor = clienți care nu primesc răspuns fără nicio urmă.

## Technical Description
În `docker-compose.yml` (dev și prod), serviciul redis primește: `command: redis-server --requirepass ${REDIS_PASSWORD} --appendonly yes --appendfsync everysec --maxmemory-policy noeviction --save ""` + volum named pentru `/data` + FĂRĂ `ports:` publicat (doar rețeaua internă compose; în dev, dacă ai nevoie de redis-cli, `docker compose exec`). `noeviction` e obligatoriu: pe stream-uri, evicția sub presiune de memorie = pierdere tăcută de mesaje — preferăm eroare explicită la XADD (webhook-ul răspunde 503, Meta reîncearcă, NX-51 deduplică).

## Principii CLAUDE.md aplicabile
P6 (niciodată pierdere tăcută), P7 (niciun serviciu fără autentificare).

## Implementation Steps
1. `.env` + `.env.example`: `REDIS_PASSWORD` (generat, 32+ chars) și `REDIS_URL=redis://:${REDIS_PASSWORD}@redis:6379/0`.
2. Compose: command-ul de mai sus, named volume `redis-data:/data`, scoate orice `ports:`.
3. Healthcheck compose: `redis-cli -a $REDIS_PASSWORD ping`.
4. Verifică toți clienții (webhook, worker, dispatcher, scheduler) că citesc `REDIS_URL` cu parolă.
5. Test manual documentat în PR: XADD 5 mesaje → `docker compose restart redis` → XLEN identic, consumer continuă.

## Test Cases
**Happy:** pipeline-ul merge identic cu parolă. **Edge:** memorie plină simulată (`maxmemory 10mb` temporar + flood) → XADD întoarce eroare, webhook răspunde 503, nimic evicted. **Failure:** parolă greșită în .env → serviciile pică la boot cu eroare clară de auth, nu mai târziu.
