# T002 — Branch protection pe main
**Owner:** S · **Faza:** MVP · **Zi:** Z1 · **Branch:** — (setare GitHub, nu cod) · **Complexitate:** XS · **Estimare:** 0.5h

## Goal
Face fizic imposibil push-ul direct pe `main`. Regula 1 din CONTRIBUTING devine constrângere de platformă, nu disciplină.

## Business Context
La 2 oameni + Claude Code generând cod rapid, singura plasă reală împotriva regresiilor pe producție e ca `main` să primească doar cod trecut prin PR + CI.

## Technical Description
GitHub → Settings → Branches → Add rule pe `main`: Require a pull request before merging (0 approvals required global — review-ul asimetric se forțează prin CODEOWNERS în T003), Require status checks to pass (se bifează `lint` și `test` după T004/T005), Do not allow bypassing the settings, Automatically delete head branches (în Settings → General).

## Principii CLAUDE.md aplicabile
Susține întreg modelul de lucru din CONTRIBUTING (reguli 1, 4, 10).

## Implementation Steps
1. Creează rule-ul cu require PR.
2. După merge-ul T004/T005: revino și bifează required checks `lint`, `test`.
3. Activează auto-delete branches.
4. Test: încearcă `git push origin main` direct → respins.

## Definition of Done
- [ ] Push direct pe main respins cu eroare de protecție
- [ ] PR cu CI roșu nu are buton de merge activ (verificat după T005)
- [ ] Branch-urile se șterg automat la merge

## Test Cases
**Happy:** PR verde se poate merge. **Edge:** adminul (tu) NU poate bypass-ui (setarea "do not allow bypassing"). **Failure:** push direct → `protected branch hook declined`.
