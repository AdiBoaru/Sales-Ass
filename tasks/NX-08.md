# NX-08 — Inbox-lite pe Slack pentru handoff (reply + return-to-bot)
**Owner:** S · **Faza:** P1 · **Ord:** O1 · **Branch:** feat/NX-08-slack-inbox · **Complexitate:** M · **Estimare:** 12h

## Goal
Când botul escaladează, operatorul clientului vede conversația într-un thread Slack și poate RĂSPUNDE de acolo — handoff-ul devine funcțional fără să aștepte dashboard-ul din P2.

## Business Context
Azi handoff-ul e o fundătură: botul tace (corect), dar clientul final nu primește răspuns de la nimeni dacă operatorul nu stă în WhatsApp Business app. Shadow mode (T253) și onboarding-ul clientului 1 depind de o mecanică reală de răspuns. Slack e gratuit, clientul îl are sau îl instalează în 10 minute.

## Technical Description
App Slack (bot token per workspace; canalul per business în `businesses.settings.slack_channel_id`). **Outbound (handoff→Slack):** la evenimentul de handoff, postează mesaj-părinte (client, telefon mascat, motiv, link profil) + ultimele 8 mesaje ca reply-uri în thread; salvează `slack_thread_ts` în `conversations.metadata`. **Inbound (Slack→client):** Events API pe `message` în thread-urile cunoscute — orice mesaj al unui om în thread devine reply: INSERT în outbox cu idempotency_key pe `slack_event_id` → dispatcherul existent îl trimite pe WhatsApp; `handoff_until` se prelungește. **Comenzi:** `/nx done` în thread → handoff închis, botul reia; `/nx note <text>` → doar notă internă, nu se trimite. Securitate: verificarea semnăturii Slack (`X-Slack-Signature`, timestamp ±5min) pe ambele endpointuri; mapare thread→conversation DOAR din DB, niciodată din textul mesajului.

## Principii CLAUDE.md aplicabile
P2 (canal Slack per business — izolat) · P9 (idempotență pe slack_event_id — Slack retrimite evenimente) · P7 (semnătură verificată).

## Implementation Steps
1. Manifest app Slack (scopes: chat:write, commands, channels:history unde e invitat) + doc de instalare pentru client în `docs/slack_setup.md`.
2. `src/integrations/slack.py`: client minimal (post message, post in thread) cu retry.
3. Hook în fluxul de handoff (T118): compune părintele + istoricul, salvează thread_ts.
4. `src/webhook/slack.py`: endpoint Events API (url_verification challenge + message events) și endpoint slash commands; ambele cu verificare de semnătură.
5. Reply path: dedupe pe event_id → outbox insert → event `handoff_human_reply`.
6. `/nx done`: setează handoff_until=now, event `handoff_closed_by_human`, confirmare în thread.
7. Teste cu payload-uri Slack înregistrate (semnături valide/invalide, retry duplicat).

## Definition of Done

## Test Cases
**Happy:** ciclu complet escaladare→reply→done. **Edge:** Slack retrimite același event de 3× → un singur mesaj la client. **Failure:** token Slack revocat → postarea pică, alertă către agenție, handoff-ul rămâne activ (clientul e sunat — runbook), nimic nu crapă în pipeline.
