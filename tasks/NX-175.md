# NX-175 — P1: FAQ retur — top-k + reranking pe calificatori + curățare seed

## Context
Audit live 2026-07-17 — **date confirmate independent** pe DB live (32 FAQ-uri, 8 despre retur).

Input: „Cum pot face un retur?" → botul a ales „Pot returna un produs desfăcut?"
(similarity ≈ 0.619) în locul procedurii generale „Cum returnez un produs?" (≈ 0.592).
Reply-ul începe cu „Nu." (răspunde la excepție, nu la întrebare).

Confirmat în DB (`business_id=6098812a-…`):
- **typo real**: „…returnate în 14 zile. calendaristice." — punct în mijlocul frazei;
- **duplicat real**: „Pot returna un produs cosmetic desigilat sau deschis?" și „Pot returna un
  produs desfăcut?" au **răspuns identic**, byte-cu-byte → două vecine care se bat pe același query;
- procedura generală există și e corectă: „Cum returnez un produs?" (OUG 34/2014, 14 zile).

Cauza: lookup-ul acceptă **top-1 orb** pe cosine. Întrebarea generică e semantic aproape de
excepție („retur" + „produs"), iar duplicatul îi dublează masa. Fără reranking pe **calificatori**
(desfăcut/desigilat/deschis), excepția câștigă în fața procedurii.

## Definition of Done
- [ ] Lookup **top-k**, nu acceptare oarbă a lui top-1.
- [ ] Reranking (lexical/semantic sau reguli) pe calificatori: „desfăcut/desigilat/deschis/folosit".
- [ ] Query generic („cum fac retur", „cum pot face un retur") → **procedura generală**.
- [ ] Query explicit despre produs desfăcut → **excepția**.
- [ ] Ambiguitate reală → **clarificare** (P6: nu tăcere, nu ghicit).
- [ ] Typo „14 zile. calendaristice." reparat + duplicatele eliminate prin **seed/backfill
      idempotent** (dry-run + audit), nu prin UPDATE manual pe live.
- [ ] Teste **unitare** (rerank pur, pe fixture) **și live** pentru ambele formulări.
- [ ] Izolare `business_id` + `locale` în orice lookup (P11: limba e parte din cheie).
- [ ] Gate verde.
