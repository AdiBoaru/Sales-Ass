## Goal
O singură unealtă principală `search_entities(query_spec)` care face TOT intern și întoarce
candidați + dovezi + acoperirea constrângerilor — faza cu cel mai mare câștig pe query-uri grele.

## Scope
1. **Pipeline intern** (audit + extindere a hibridului EXISTENT din catalog_tools.py, nu rescriere):
   exact name/SKU/alias (rapidfuzz cu praguri: mare→resolve, mediu→clarify, mic→not_found) →
   FTS ponderat → embedding pe raw query + embedding pe query canonicalizat (dual, D6) →
   filtre SQL hard → RRF → excluderi/penalizări not_recommended_for (D9) → rerank ADAPTIV
   (skip pe SKU exact; rulează când rezultatele-s apropiate / multe constrângeri / lexical vs
   semantic dezacord) → evidence hydration BATCH → constraint coverage.
2. **Output:** `{candidates, constraint_coverage[{facet, status: MATCH|NO_MATCH|UNKNOWN}],
   missing_information, needs_refinement, evidence}` — UNKNOWN ≠ NO_MATCH (D7).
   Semantica vine din NX-187: mulțimi DISJUNCTE cu precedență (rejected → alternatives → exact);
   **soft constraints influențează DOAR scorul, nu apartenența**. Tipurile/operatorii/politica
   `missing_value` vin din registrul NX-186.
3. **Selecția furnizorilor pe benchmark NX-203:** embeddings (actual vs 2 alternative) ×
   reranker (fără vs 2 API-uri) — câștigă cifrele, nu faima. Reranker prin API (nu local — infra).
4. Operațiile independente rulează în PARALEL (D12).

## Gate (ținte inițiale)
Recall@20 ≥90% (apoi ≥95%) · produs relevant în Top-6 ≥90% · nDCG@6 ≥0,85 · 0 încălcări hard confirmate.

## Out of Scope
Folosirea de către agentul nou (NX-210). Schimbarea tool-urilor existente în prod (shadow până la F7).

## Definition of Done
- [ ] `search_entities` funcțional cu contractul de output complet; consumă registrul NX-186
      (nu redefinește tipuri/operatori) și semantica MatchSet din NX-187.
- [ ] Benchmark embeddings×reranker publicat; alegerea justificată cu cifre (nu cu faima furnizorului).
- [ ] Gate-ul de retrieval atins pe NX-203 (Recall@20 ≥90%, nDCG@6 ≥0,85, Top-6 ≥90%, 0 hard violations).
- [ ] Latență măsurată per componentă (spans NX-201); rerank adaptiv demonstrat (NU rulează pe SKU exact).
- [ ] Invariante: `WHERE business_id = $1` explicit pe toate query-urile noi + test adversarial
      cross-tenant; `business_id` server-owned, niciodată din output-ul modelului; zero PII către
      serviciul de rerank (politica NX-200, cu opt-out per business); **degradare fără reranker/
      embeddings noi** (fațetă/serviciu indisponibil → cale veche, fără regresie, fără tăcere — P6);
      kill-switch per componentă; `ruff` + `ruff format --check` + `pytest -x -q` verzi.
